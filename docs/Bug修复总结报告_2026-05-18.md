## 🐛 Bug #1: 训练时间范围生成失败

### 问题表现

- 用户打开应用后，训练日期无法生成
- 页面持续转圈，无响应
- 控制台报错：`FormatException: Invalid radix-10 number (at character 1) 2005-01-04`

### 根本原因

数据库中的 `trade_date` 字段存储为字符串格式（如 "2005-01-04"），但 Drift ORM 模型中定义为 `DateTimeColumn`，导致 Drift 尝试将字符串解析为整数时间戳，抛出格式异常。

### 解决方案

1. **添加日期转换辅助方法**
   - `_dateToString()`: DateTime → "YYYY-MM-DD"
   - `_stringToDate()`: "YYYY-MM-DD" → DateTime

2. **使用自定义SQL查询**
   - 所有日期相关查询改用 `customSelect()`
   - 手动处理日期字符串与 DateTime 对象之间的转换

3. **手动构建数据对象**
   - 添加 `_buildKlineDataFromRow()` 方法
   - 从 QueryRow 手动构建 KlineDataData 对象

### 修改的文件

- `lib/data/database/daos/stock_filter_dao.dart`
- `lib/data/database/daos/kline_dao.dart`

### 测试结果

✅ **成功**：训练日期成功生成（2023-11-20）

## 🐛 Bug #2: 选股结果始终为0

### 问题表现

- 用户选择条件后（如A股、上证&深证、200日新高），未查询出结果
- 页面显示"请选择条件查询结果"
- 控制台报错：`Null check operator used on a null value`

### 根本原因

发现了两个关键问题：

#### 问题1：状态管理不同步

存在两个独立的状态管理：
- `selectionProvider` - 管理市场选择（`selectedSubMarkets`）
- `stockFilterProvider` - 管理筛选逻辑（`selectedMarkets`）

`MarketSelector` 更新的是 `selectionProvider`，但筛选使用的是 `stockFilterProvider`，两者不同步。

#### 问题2：数据库模型与实际表结构不匹配

Drift 模型中定义了 `lotSize` 和 `minTick` 字段，但实际数据库中的 `symbols` 表没有这些字段。当 Drift 尝试映射数据时，这些字段返回 null，但模型中使用了 `.withDefault()`，Drift 会强制解包这些值，导致空值错误。

### 解决方案

#### 解决问题1：同步状态管理

1. 在 `selection_provider.dart` 中，让 `setSubMarkets` 方法调用 `_executeStockFilter()` 触发筛选
2. 在 `stock_condition_selector.dart` 中，让 `_onConditionSelected` 方法同步市场选择到 `stockFilterProvider`

#### 解决问题2：修复数据库模型

将 `lotSize` 和 `minTick` 字段从 `.withDefault()` 改为 `.nullable()`，避免强制解包空值。

### 修改的文件

- `lib/data/database/tables/symbols_table.dart`
- `lib/providers/selection_provider.dart`
- `lib/features/home/widgets/stock_condition_selector.dart`

### 测试结果

✅ **成功**：应用成功启动，空值错误已修复
