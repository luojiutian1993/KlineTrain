# 训练页面返回卡顿与全量刷新问题修复报告

## 1. 问题概述

### 1.1 问题现象
- **触发场景**：用户在训练页面（`TrainingScreen`）点击"详情"进入训练详情页
- **预期结果**：点击返回按钮后立即返回训练页面，保持当前状态
- **实际结果**：返回时出现卡顿，且页面发生全量刷新（重新初始化）

### 1.2 问题影响范围
- 训练页面（`TrainingScreen`）
- 路由导航（`app_routes.dart`）
- 主页面组件（`MainTabPage`）

---

## 2. 根本原因分析

### 2.1 问题根因

**问题1：`didChangeDependencies` 中的数据加载**
- **文件**：[training_screen.dart](file:///Users/lin/dev/KlineTrain/lib/features/training/training_screen.dart#L28-L29)
- **问题**：在 `didChangeDependencies()` 中调用 `_loadTrainingSessions()`，导致每次返回页面时都会重新加载数据
- **影响**：页面每次从详情页返回都会重新初始化

**问题2：路由 Key 使用时间戳**
- **文件**：[app_routes.dart](file:///Users/lin/dev/KlineTrain/lib/routes/app_routes.dart#L103)
- **问题**：`MainTabPage` 的 Key 中使用了 `DateTime.now().millisecondsSinceEpoch`，导致每次导航都创建新实例
- **影响**：无法复用已缓存的页面组件

### 2.2 问题架构图

```
用户操作流程：
┌─────────────────────────────────────────────────────────────┐
│  训练页面 (TrainingScreen)                                  │
│       │                                                     │
│       ▼ 点击"详情"                                          │
│  路由导航 → 创建 TrainingDetailScreen                       │
│       │                                                     │
│       ▼ 点击返回                                            │
│  didChangeDependencies() 被触发                             │
│       │                                                     │
│       ├─ 问题1: _loadTrainingSessions() 被调用              │
│       └─ 问题2: MainTabPage Key 包含时间戳导致重建           │
│       │                                                     │
│       ▼                                                     │
│  页面重新初始化 + 卡顿                                      │
└─────────────────────────────────────────────────────────────┘
```

---

## 3. 修复方案

### 3.1 修复内容

#### 修复1：移除 didChangeDependencies 中的数据加载

**文件**：[training_screen.dart](file:///Users/lin/dev/KlineTrain/lib/features/training/training_screen.dart#L28-L30)

```diff
  @override
  void didChangeDependencies() {
    super.didChangeDependencies();
-   _loadTrainingSessions();
  }
```

**修复原理**：
- `didChangeDependencies` 在依赖变化时会被调用
- 导航返回时依赖可能变化，导致重复加载
- 数据加载仅需在 `initState` 时执行一次

#### 修复2：移除路由 Key 中的时间戳

**文件**：[app_routes.dart](file:///Users/lin/dev/KlineTrain/lib/routes/app_routes.dart#L99-L107)

```diff
  if (tabIndex == 1 && symbol.isNotEmpty) {
    final battleParams = BattleScreenParams(...);
-   return MainTabPage(
-     key: ValueKey(
-       'main_${symbol}_${sessionId ?? 0}_${isReplayMode ? 'replay' : 'normal'}_${DateTime.now().millisecondsSinceEpoch}'),
+   final stableKey = 'main_${symbol}_${sessionId ?? 0}_${isReplayMode ? 'replay' : 'normal'}';
+   return MainTabPage(
+     key: ValueKey(stableKey),
    );
  }
- return MainTabPage(initialIndex: tabIndex);
+ return const MainTabPage(initialIndex: 0);
```

**修复原理**：
- 使用稳定的 Key 确保相同参数创建相同的组件
- 移除时间戳避免每次路由都创建新实例

#### 修复3：添加 AutomaticKeepAliveClientMixin

**文件**：[training_screen.dart](file:///Users/lin/dev/KlineTrain/lib/features/training/training_screen.dart#L15-L26)

```dart
class _TrainingScreenState extends ConsumerState<TrainingScreen>
    with AutomaticKeepAliveClientMixin {

  @override
  bool get wantKeepAlive => true;

  @override
  Widget build(BuildContext context) {
    super.build(context);
    return Scaffold(...)
  }
}
```

**修复原理**：
- `AutomaticKeepAliveClientMixin` 确保页面状态在切出时保留
- `wantKeepAlive: true` 防止页面被系统回收
- `super.build(context)` 必须在 build 方法中调用
