# 多账户系统与默认账号功能需求文档

## 1. 概述

### 1.1 功能背景

随着产品迭代，个人页面体系持续变化，为满足测试和演示需求，需要：
1. 设定默认测试账号 `13071103531`，免登录进入应用
2. 实现多账户体系，确保各账户数据完全隔离
3. 保证训练页面记录、个人信息等与数据库一致，不影响原有功能
4. 支持账号注册、登录、密码找回等完整账号体系

### 1.2 设计原则

- **默认账号优先**：账号 `13071103531` 作为默认测试账号，应用启动时自动登录
- **数据隔离**：所有训练记录、持仓、资金等数据严格按账户隔离
- **状态保持**：账号切换后，原账号信息保留，下次登录可继续训练
- **资金统一**：全局默认初始资金为 **100000 元**，不可配置
- **账号安全**：支持密码验证、重复注册检测、密码重置

## 2. 核心需求

### 2.1 默认账号登录机制

| 需求项 | 描述 |
|--------|------|
| 默认账号 | `13071103531` |
| 登录方式 | 静默自动登录，用户无感知 |
| 账号来源 | 数据库已存在该账号，数据需正常展示 |
| 退出入口 | "我的"页面提供"退出登录"按钮 |
| 新账号注册 | 退出后支持注册新账号，进入新账号体系 |

**用户流程**：
```
应用启动
    ↓
检测默认账号 13071103531
    ↓
静默自动登录 → 进入首页（默认账号体系）
    ↓
点击"我的" → 可查看个人数据
    ↓
点击"退出登录" → 退出到登录页面
    ↓
注册新账号 / 登录已有账号 → 进入对应账号体系
```

### 2.2 账户数据隔离

**隔离范围**：

| 数据类型 | 隔离方式 | 说明 |
|----------|----------|------|
| 训练记录 | 按 userId 隔离 | 各账户只能查看自己的训练记录 |
| 持仓信息 | 按 userId 隔离 | 各账户只能操作自己的持仓 |
| 资金数据 | 按 userId 隔离 | 各账户资金独立计算 |
| 用户信息 | 按 userId 隔离 | 昵称、头像等个人信息独立 |
| 条件单 | 按 userId 隔离 | 各账户条件单独立 |

**数据关联顺序**：
```
先有账户信息
    ↓
再有训练信息（训练记录、持仓、条件单等）
    ↓
账户切换时保留原账户数据
```

### 2.3 默认初始资金

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 初始资金 | 100000 元 | 全局固定，不可配置 |
| 适用范围 | 所有账户 | 默认账号、新注册账号统一 |

### 2.4 账号注册与登录

| 需求项 | 描述 |
|--------|------|
| 手机号唯一性 | 同一手机号不能重复注册 |
| 重复注册检测 | 注册前检查手机号是否已存在 |
| 密码登录 | 用户名（手机号）+ 密码登录 |
| 忘记密码 | 支持密码重置功能 |

**注册流程**：
```
输入手机号、验证码、密码
    ↓
系统检查手机号是否已注册
    ↓
已注册 → 提示"该手机号已注册，请直接登录或找回密码"
    ↓
未注册 → 创建新账号 → 自动登录 → 进入首页
```

**登录流程**：
```
输入手机号、密码
    ↓
验证密码是否正确
    ↓
正确 → 登录成功 → 进入首页（显示该账号数据）
    ↓
错误 → 提示"手机号或密码错误"
```

**密码重置流程**：
```
点击"忘记密码"
    ↓
输入手机号、验证码、新密码
    ↓
验证通过 → 重置密码成功 → 跳转登录页
```

### 2.5 我的页面数据展示

**数据来源**：
- 个人页面展示当前登录账户的数据
- 训练统计从数据库按 userId 查询

**展示内容**：

| 字段 | 数据来源 | 说明 |
|------|----------|------|
| 昵称 | users 表（当前用户） | 可编辑 |
| 手机号 | users 表（当前用户） | 脱敏显示 |
| 会员等级 | users 表（当前用户） | 基于训练数据计算 |
| 训练次数 | training_sessions 表 | 按 userId 统计 |
| 总收益率 | training_sessions 表 | 按 userId 统计 |
| 交易胜率 | training_sessions 表 | 按 userId 统计 |
| 学习进度 | 用户学习记录表 | 按 userId 统计 |

## 3. 数据库设计

### 3.1 现有表结构（已满足需求）

**users 表**：存储用户基本信息
```sql
- id: 用户ID（主键）
- phone: 手机号（唯一索引）
- nickname: 昵称
- member_level: 会员等级
- created_at: 创建时间
```

**user_passwords 表**：存储用户密码
```sql
- id: 记录ID（主键）
- user_id: 用户ID（外键）
- password_hash: 密码哈希
- created_at: 创建时间
- updated_at: 更新时间
```

**training_sessions 表**：存储训练记录
```sql
- id: 会话ID（主键）
- user_id: 用户ID（外键）
- symbol: 标的代码
- initial_capital: 初始资金
- current_capital: 当前资金
- total_profit: 总盈亏
- profit_rate: 收益率
- ...
```

### 3.2 查询逻辑

```dart
// 获取当前用户训练记录
Future<List<TrainingSession>> getUserSessions(int userId) {
  return (select(trainingSessions)
    ..where((t) => t.userId.equals(userId))
    ..orderBy([(t) => OrderingTerm.desc(t.createdAt)]))
    .get();
}

// 检查手机号是否已注册
Future<UserModel?> getUserByPhone(String phone) {
  return (select(users)
    ..where((u) => u.phone.equals(phone)))
    .getSingleOrNull();
}

// 验证密码
Future<bool> verifyPassword(String phone, String password) {
  final user = await getUserByPhone(phone);
  if (user == null) return false;
  return passwordHash.verify(password, user.passwordHash);
}
```

## 4. 验收标准（AC）

### 4.1 默认账号登录

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-001 | Given 应用启动，When 数据库中存在 13071103531 账号，Then 自动静默登录，用户直接进入首页 | Happy Path |
| AC-002 | Given 默认账号登录，When 用户访问"我的"页面，Then 展示该账号的历史训练记录和个人信息 | Happy Path |
| AC-003 | Given 默认账号登录，When 用户点击"退出登录"，Then 退出到登录页面 | Happy Path |

### 4.2 账号注册

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-004 | Given 用户在注册页面，When 输入未注册的手机号和新密码，Then 创建新用户并自动登录，数据从零开始 | Happy Path |
| AC-005 | Given 用户在注册页面，When 输入已注册的手机号，Then 显示"该手机号已注册，请直接登录或找回密码" | Edge Case |
| AC-006 | Given 注册成功，When 跳转首页，Then "我的"页面显示新账号的昵称" K线新手"和手机号 | Happy Path |

### 4.3 账号登录

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-007 | Given 用户在登录页面，When 输入正确的手机号和密码，Then 登录成功，跳转首页，显示该账号数据 | Happy Path |
| AC-008 | Given 用户在登录页面，When 输入错误的密码，Then 显示"手机号或密码错误" | Edge Case |
| AC-009 | Given 用户在登录页面，When 输入未注册的手机号，Then 显示"手机号或密码错误" | Edge Case |

### 4.4 密码重置

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-010 | Given 用户点击"忘记密码"，When 输入已注册手机号、验证码、新密码，Then 密码重置成功，跳转登录页 | Happy Path |
| AC-011 | Given 用户点击"忘记密码"，When 输入未注册手机号，Then 提示"该手机号未注册" | Edge Case |

### 4.5 账户数据隔离

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-012 | Given 账号 A 进行训练，When 切换到账号 B，Then 账号 A 的训练记录、持仓、资金完全保留 | Happy Path |
| AC-013 | Given 账号 B 登录，When 查看训练记录，Then 仅显示账号 B 的训练记录，不显示账号 A 的 | Edge Case |
| AC-014 | Given 账号 A 持仓某股票，When 账号 B 查看持仓，Then 账号 B 持仓为空 | Edge Case |
| AC-015 | Given 多账号切换，When 再次登录账号 A，Then 账号 A 的状态与上次退出时一致 | Happy Path |
| AC-016 | Given 账号 A 设置了条件单，When 切换到账号 B，Then 账号 B 的条件单为空或独立 | Edge Case |

### 4.6 训练功能

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-017 | Given 默认账号登录，When 进入实战页面，Then 初始资金显示为 100000 元 | Happy Path |
| AC-018 | Given 任何账号新训练，When 开始训练，Then 初始资金为 100000 元 | Business Rule |
| AC-019 | Given 账号进行买入交易，When 查看资金变化，Then 资金正确扣减，持仓正确增加 | Happy Path |
| AC-020 | Given 账号完成训练，When 训练结束，Then 训练记录保存到当前账号的数据库 | Happy Path |
| AC-021 | Given 训练结束，When 查看训练记录详情，Then 复盘数据与训练过程一致 | Happy Path |

### 4.7 我的页面

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-022 | Given 账号登录，When 查看"我的"页面，Then 展示当前账号的昵称、手机号、等级 | Happy Path |
| AC-023 | Given 账号有训练记录，When 查看统计数据，Then 训练次数、收益率、胜率等数据来自当前账号 | Happy Path |
| AC-024 | Given 账号无训练记录，When 查看统计数据，Then 显示 0 或默认值 | Edge Case |
| AC-025 | Given 账号切换，When 查看"我的"页面，Then 显示切换后账号的数据，非切换前账号 | Edge Case |

### 4.8 换股功能

| 编号 | 验收条件 | 场景类型 |
|------|----------|----------|
| AC-026 | Given 账号 A 训练中换股，When 账号 B 查看换股结果，Then 账号 B 不受影响 | Edge Case |
| AC-027 | Given 账户信息未加载，When 用户尝试交易，Then 提示"请先选择股票"，不执行交易 | Edge Case |

## 6. 关键实现点

### 6.1 静默登录逻辑

```dart
// main_tab_page.dart
Future<void> _initAuth() async {
  if (_initialized) return;
  _initialized = true;

  final authState = ref.read(authStateProvider);
  if (!authState) {
    final success = await ref.read(authStateProvider.notifier).silentLogin();
    if (!success && mounted) {
      context.go(AppRoutes.login);
    }
  }
}
```

### 6.2 重复注册检测

```dart
// auth_provider.dart - register 方法
Future<bool> register(String phone, String password, String email) async {
  try {
    final userRepository = UserRepository();
    
    final existingUser = await userRepository.getUserByPhone(phone);
    if (existingUser != null) {
      throw Exception('该手机号已注册，请直接登录或找回密码');
    }

    final newUser = UserModel(...);
    await userRepository.saveUserToDatabase(newUser, password: password);
    state = true;
    ref.read(currentUserNotifierProvider.notifier).setUser(newUser);
    return true;
  } catch (e) {
    rethrow;
  }
}
```
