## 1. 背景与价值

**为什么做**：当前K线图表的交互方式有限，仅支持按钮操作，缺少触屏手势支持，用户体验不够流畅。同时缩放和滑动的逻辑存在边界处理问题，需要优化交互体验。

**做成什么样**：支持触屏滑动和双指缩放，优化缩放逻辑保持当前日期在可视区域，增加边界防抖机制，完善用户交互体验。

**关联**：项目核心功能优化 — 提升训练页面用户交互体验

---

## 2. 范围

### 本次做

- 支持触屏左右滑动查看历史K线
- 支持双指缩放操作（放大缩小）
- 优化+-按钮的放大缩小功能
- 缩小最多支持显示700根K线
- 缩放时保持最右侧当前日期在可视区域
- 边界防抖机制，到达边界时只提示一次
- 右滑限制在当前训练日线范围内，不显示未训练数据
- 下方指标图表（成交量、MACD等）跟随K线滑动和缩放同步变化

### 本次不做

- 不修改K线数据获取逻辑
- 不修改交易功能核心逻辑
- 不修改K线图表渲染样式

## 4. 需求详述

### 4.1 核心流程

1. 用户进入实战页面
2. 用户可以通过以下方式操作K线图：
   - 点击+-按钮放大缩小
   - 触屏左右滑动查看历史数据
   - 双指捏合/张开进行缩放
3. 所有操作都保持当前日期在可视区域右侧
4. 下方指标图表（成交量、MACD等）同步跟随变化
5. 到达边界时显示提示信息（防抖）
6. 右滑不会超过当前训练日线

### 4.2 交互规则

- **触屏滑动**：水平滑动K线图表区域可以左右移动可视范围
- **双指缩放**：双指捏合缩小，双指张开放大
- **按钮缩放**：点击+放大（减少显示K线数量），点击-缩小（增加显示K线数量）
- **边界限制**：
  - 左滑最多到第0根K线
  - 右滑最多到当前训练日线（currentDayIndex）
  - 缩小最多显示700根K线
  - 放大最少显示10根K线
- **当前日期保持**：缩放操作时，当前日期（currentDayIndex对应的K线）始终保持在可视区域的最右侧
- **边界提示**：到达边界时显示提示信息，同一边界在短时间内多次操作只提示一次
- **指标同步**：下方的成交量、MACD等指标图表必须与主K线图保持同步，显示相同时间范围的数据

## 5. 验收标准（AC）

### 5.1 正常流程（Happy Path）

- **AC-001**: Given 用户在实战页面且K线图表已加载，When 用户在K线图表区域水平左滑，Then K线图向左移动，显示更早的历史数据，指标图表同步变化。
- **AC-002**: Given 用户在实战页面且K线图表已加载，When 用户在K线图表区域水平右滑，Then K线图向右移动，显示更近的数据，指标图表同步变化。
- **AC-003**: Given 用户在实战页面且K线图表已加载，When 用户双指捏合（向内滑动），Then K线图缩小，显示更多K线，最多不超过700根，当前日期保持在最右侧，指标图表同步变化。
- **AC-004**: Given 用户在实战页面且K线图表已加载，When 用户双指张开（向外滑动），Then K线图放大，显示更少K线，最少不少于10根，当前日期保持在最右侧，指标图表同步变化。
- **AC-005**: Given 用户在实战页面且K线图表已加载，When 用户点击+按钮，Then K线图放大，显示K线数量减少，当前日期保持在最右侧，指标图表同步变化。
- **AC-006**: Given 用户在实战页面且K线图表已加载，When 用户点击-按钮，Then K线图缩小，显示K线数量增加，最多不超过700根，当前日期保持在最右侧，指标图表同步变化。
- **AC-007**: Given 用户在实战页面且K线图表已加载，When 用户进行任何滑动或缩放操作，Then 下方的成交量、MACD等指标图表始终与主K线图显示相同时间范围的数据。

### 5.2 边界与异常（Edge & Error Cases）

- **AC-008**: Given 用户已滑动到最左侧（第0根K线），When 用户继续左滑或点击左滑按钮，Then 显示"已到达最左边"提示，且在3秒内重复操作不重复提示。
- **AC-009**: Given 用户已滑动到最右侧（当前训练日线），When 用户继续右滑或点击右滑按钮，Then 显示"已到达最右边"提示，且在3秒内重复操作不重复提示。
- **AC-010**: Given 当前已显示最多700根K线，When 用户继续点击-按钮或双指捏合缩小，Then 不继续缩小，显示"已达到最小缩放级别"提示。
- **AC-011**: Given 当前已显示最少10根K线，When 用户继续点击+按钮或双指张开放大，Then 不继续放大，显示"已达到最大缩放级别"提示。
- **AC-012**: Given 用户右滑查看历史数据后，When 用户点击"下一步"推进到下一个交易日，Then K线图自动调整，新的当前日期显示在最右侧。

### 5.3 业务规则（Business Rules）

- **AC-013**: Given 任何缩放操作（按钮或双指），When 缩放完成，Then 当前训练日线（currentDayIndex对应的K线）始终显示在可视区域的最右侧。
- **AC-014**: Given 任何滑动操作，When 滑动完成，Then 可视范围的最右侧不超过当前训练日线（currentDayIndex）。
- **AC-015**: Given 任何缩放操作，When 计算显示K线数量，Then 数量必须在10-700根范围内。
- **AC-016**: Given 任何边界提示，When 同一方向在3秒内多次触发，Then 只显示一次提示信息。

## 6. 非功能性需求

- **响应速度**：滑动和缩放操作响应时间不超过100ms
- **流畅度**：滑动和缩放动画保持60fps
- **稳定性**：连续快速操作不导致应用崩溃
