# 实战页面布局优化 — 功能需求说明书

## 1. 背景与价值

**为什么做**：当前实战页面在不同屏幕尺寸下布局比例不协调，存在部分区域显示拥挤或空白过多的问题。用户在训练过程中需要同时查看K线图和指标图，布局比例不合理会影响分析效率。

**做成什么样**：实现自适应不同手机屏幕的响应式布局，K线区域与两个指标区域比例为2:1:1，字体大小统一规范，整体比例协调，无滑动框。

**关联**：项目核心功能优化

---

## 2. 范围

### 本次做

- 调整布局结构，确保K线区域:指标区域1:指标区域2 = 2:1:1
- 移除所有滑动框（SingleChildScrollView），实现单屏显示
- 统一页面各区域字体大小和样式格式
- 确保布局自适应不同手机屏幕尺寸（从小屏手机到大屏手机）
- 确保iOS设备使用SafeArea避免notch和屏幕边缘遮挡

### 本次不做

- 不修改指标计算逻辑
- 不修改交易功能逻辑
- 不修改K线图表渲染逻辑
- 不修改底部导航栏结构

## 4. 需求详述

### 4.1 核心流程

1. 用户进入实战页面
2. 页面自适应设备屏幕尺寸，使用SafeArea包裹
3. 各区域按比例分配空间，无滑动框

### 4.2 交互规则

- 页面加载完成后所有内容一屏显示，无需上下滚动
- 各区域高度根据屏幕尺寸自动调整
- 字体大小根据区域类型统一规范

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | K线区域高度 : 指标区域1高度 : 指标区域2高度 = 2 : 1 : 1 |
| BR-002 | 页面必须使用SafeArea包裹，避免iOS notch和屏幕边缘遮挡 |
| BR-003 | 页面不得包含SingleChildScrollView，所有内容一屏显示 |
| BR-004 | 股票信息区域标签字体大小统一为10号，数据字体大小统一为11号 |
| BR-005 | 控制按钮区域字体大小统一为10号 |
| BR-006 | 指标选择器字体大小统一为10号 |
| BR-007 | 交易按钮区域字体大小统一为10号 |
| BR-008 | 资产信息区域标签字体大小统一为9号，数据字体大小统一为10号 |
