## 1. 任务概览

**总任务数**：22 个
**预计总工时**：约 40-60 小时（按阶段分布）
**开发方法**：TDD + 分阶段重构 — 每个任务按 RED → GREEN → REFACTOR 循环执行

**关键标注**：
- 🔒 阻塞任务：被多个任务依赖，建议优先完成
- ⚠️ 风险任务：技术难度高，可能需要额外时间

## 2. 开发任务

### 阶段一：基础设施层

**阶段完成标准**：新建 Provider/Model 可独立编译，不影响现有功能

---

#### Task-1.1: 创建 BattleState 状态模型 🔒

**通俗解释**：定义训练状态的"蓝图"，让所有组件知道有哪些数据可以共享

**做什么**：
1. 在 `lib/features/battle/models/` 目录创建 `battle_state.dart`
2. 将现有 `TrainingState` 字段完整迁移
3. 添加派生属性（`currentKline`、`positionValue`、`canTrade` 等）
4. 实现 `copyWith` 方法

**涉及文件**：
- 新建：`lib/features/battle/models/battle_state.dart`
- 参考：`lib/features/battle/training_state.dart`

**参考**：技术方案 4.2 → AC-ARCH-001

**依赖**：无

**预估工时**：60 分钟

**验证标准**（TDD RED 阶段直接转化为测试用例）：
- [ ] `BattleState()` 创建默认状态，所有字段有默认值
- [ ] `state.copyWith(currentSymbol: 'SH600000')` 返回新实例，原实例不变
- [ ] `state.currentKline` 在 `allKlineData` 为空时返回 `null`
- [ ] `state.currentKline` 正确返回 `_currentDayIndex` 对应的 K线数据
- [ ] `state.positionValue` 等于 `positionQuantity * currentPrice`

**涉及文件**：
- 新建：`lib/features/battle/services/stock_service.dart`

**参考**：技术方案 4.3.2 → AC-FUNC-001

**依赖**：Task-1.3

**预估工时**：90 分钟

**验证标准**（TDD RED 阶段直接转化为测试用例）：

#### Task-2.2: 创建 TrainingService 训练服务 ⚠️

**通俗解释**：把训练流程（开始、暂停、重置、统计）独立出来，不依赖具体 UI

**做什么**：
1. 在 `lib/features/battle/services/` 目录创建 `training_service.dart`
2. 实现 `start()` 方法：初始化训练
3. 实现 `dayForward()` / `dayBackward()` 方法：控制训练进度
4. 实现 `pause()` / `reset()` 方法：训练控制
5. 实现 `stats` 属性：获取训练统计

**涉及文件**：
- 新建：`lib/features/battle/services/training_service.dart`
- 依赖：`lib/features/battle/providers/battle_provider.dart`

**参考**：技术方案 4.3.1 → AC-FUNC-002

**依赖**：Task-2.1

**预估工时**：90 分钟

**验证标准**（TDD RED 阶段直接转化为测试用例）：
- [ ] `start(symbol: 'SZ000021', startDate: DateTime(2023,3,31))` 设置正确的训练起始日期
- [ ] `dayForward()` 使 `currentDayIndex` 增加 1
- [ ] `dayForward()` 在 `currentDayIndex >= allKlineData.length - 1` 时不操作
- [ ] `dayBackward()` 使 `currentDayIndex` 减少 1
- [ ] `dayBackward()` 在 `currentDayIndex <= historyDays` 时不操作
- [ ] `reset()` 恢复初始状态
