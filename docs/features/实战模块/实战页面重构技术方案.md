# 实战页面重构 - 技术方案

**文档版本**: v1.0
**创建日期**: 2026-05-28
**最后更新**: 2026-05-28

## 1. 设计概要

### 1.1 问题描述

`lib/features/battle/battle_screen.dart` 文件超过 4500 行代码，承担状态管理、业务逻辑、UI 渲染等多重职责，严重违反单一职责原则（SRP）。

**职责混淆问题**：
| 职责类型 | 当前实现 | 问题 |
|---------|---------|------|
| 状态管理 | 40+ 个 State 变量 | 难以维护，状态分散 |
| 业务逻辑 | K线处理、交易计算、选股逻辑混合 | 难以测试 |
| UI渲染 | 100+ 个 Widget 构建方法 | 文件过大，难以定位 |
| 指标计算 | 11 个指标图表独立实现 | 代码重复 |
| 数据持久化 | 数据库操作内嵌 | 违反分层架构 |

### 1.2 重构目标

**核心约束**：保证原有页面格局和功能不变

**重构目标**：
1. 文件拆分：将 4500+ 行文件拆分为 10-15 个职责单一的文件
2. 状态集中：使用 Riverpod 统一管理训练状态
3. 逻辑解耦：业务逻辑与 UI 分离，提升可测试性
4. 代码复用：抽取公共组件和逻辑

### 1.3 技术决策

| 决策点 | 选项 | 选择 | 理由 |
|-------|------|------|------|
| 状态管理 | StatefulWidget / Riverpod | **Riverpod** | 与项目现有架构一致，学习成本低 |
| 模块复用 | 新建 / 复用现有文件 | **复用** | trading_calculator.dart 已实现交易计算 |
| 重构策略 | 一次性 / 分阶段 | **分阶段** | 降低风险，确保功能稳定 |
| 架构模式 | 简单拆分 / 完整MVVM | **简单分层** | 符合项目规模和复杂度 |

## 6. 关键技术决策

### 6.1 Riverpod vs StatefulWidget

**决策**：选择 Riverpod

**理由**：
1. 与项目现有架构一致（已有 `trading_provider.dart`、`kline_provider.dart`）
2. 状态集中管理，便于调试
3. 支持依赖注入，便于 Service 注入
4. 编译时安全性（`riverpod_annotation`）

**备选方案**：StatefulWidget + InheritedWidget
- 缺点：需要手动管理状态传递
- 缺点：难以实现跨模块状态共享

### 6.2 状态不可变性

**决策**：使用不可变对象（Immutable State）

```dart
// 使用 copyWith 模式
final newState = state.copyWith(
  currentDayIndex: state.currentDayIndex + 1,
  phase: TrainingPhase.closing,
);
```

**理由**：
1. 便于状态回溯和调试
2. Riverpod 的 `AsyncNotifier` 需要不可变更新
3. 避免意外状态修改

### 6.3 Widget 通信

**决策**：Widget 通过 Consumer 获取 Provider 状态，通过回调修改状态

```dart
// Widget 读取状态
final state = ref.watch(battleProvider);

// Widget 触发操作
final notifier = ref.read(battleProvider.notifier);
notifier.nextDay();
```

**替代方案**：
- Controller 模式：Widget 直接持有逻辑 → 违背单一职责
- Bloc 模式：引入额外复杂性 → 过度设计

### 4.4 指标计算位置

**决策**：保持现有位置（各 Widget 内部）

**理由**：
1. 指标计算已通过 `indicator_calculator.dart` 统一
2. 各指标 Widget 已有独立文件（`widgets/` 目录）
3. 移动到 Provider 会增加复杂度

## 4.5 缓存策略 - 指标数据缓存

**决策**：添加指标计算缓存机制

**设计背景**：
当前实现中，每次 Widget build 都会重新计算 MACD/KDJ 等指标，在大数据量或频繁刷新时可能造成性能问题。

**缓存结构**：

**缓存失效条件**：
| 触发条件 | 缓存行为 |
|---------|---------|
| K线数据变化 | 清除缓存 |

**缓存生命周期**：
```

    ↓

    ↓

    ↓

    ↓
检查缓存是否有效
    ↓
有效 → 直接返回缓存数据
无效 → 重新计算 → 更新缓存
```

## 4.6 状态恢复机制

**决策**：Provider 内存缓存 + 数据库持久化（混合方案）

**设计背景**：
用户切换 Tab 后返回，需要恢复训练进度。使用 Provider 的 `keepAlive` 机制可以保持内存状态，但应用重启后会丢失。因此需要结合数据库持久化。

**状态分层**：
| 状态类型 | 存储方式 | 生命周期 |
|---------|---------|---------|
| 当前股票 | Provider | Tab 切换期间 |
| 训练进度 | Provider + DB | 跨会话 |
| 账户信息 | Provider + DB | 跨会话 |
| K线数据 | Provider（内存缓存）| Tab 切换期间 |
| 指标缓存 | Provider（内存）| 实时重建 |

**数据库持久化策略**：
```dart
// 训练会话表（已有 training_sessions）
// 在以下时机保存到数据库：
// 1. 用户点击"下一步"推进到新的一天
// 2. 用户执行买入/卖出操作
// 3. 用户主动保存训练
// 4. 应用进入后台时

Future<void> _saveTrainingProgress() async {
  await dbService.trainingDao.updateSession(sessionId, {
    'currentDayIndex': state.currentDayIndex,
    'accountBalance': state.accountBalance,
    'positionQuantity': state.positionQuantity,
    'positionCost': state.positionCost,
    'lastUpdateTime': DateTime.now(),
  });
}
```

**状态恢复流程**：
```
应用启动 / 路由跳转
    ↓
检查是否有进行中的训练会话
    ↓
有会话ID → 从数据库加载会话状态
    ↓
恢复 Provider 状态
    ↓
加载对应的 K线数据
    ↓
显示恢复后的训练界面
```

## 4.7 刷新机制

**决策**：仅路由参数控制刷新

**设计背景**：
实战页面不需要下拉刷新，因为 K线数据是历史数据，不需要从服务器拉取最新数据。刷新由路由参数控制，例如从首页重新选股跳转。

**刷新触发场景**：
| 场景 | 刷新方式 | 说明 |
|------|---------|------|
| 从首页选股跳转 | 路由参数 | symbol/name/market 作为参数传递 |
| 重新选股 | 路由跳转 | 携带新参数跳转到 /battle |
| 复盘模式 | 路由参数 | sessionId 加载历史数据 |
| 重置训练 | Provider 方法 | 调用 `reset()` 方法 |

**路由参数定义**：
```dart
// lib/routes/app_routes.dart

GoRoute(
  path: battle,
  name: 'battle',
  builder: (context, state) {
    final symbol = state.uri.queryParameters['symbol'] ?? '';
    final name = state.uri.queryParameters['name'] ?? '';
    final market = state.uri.queryParameters['market'] ?? '';
    final dateStr = state.uri.queryParameters['date'] ?? '';
    final sessionIdStr = state.uri.queryParameters['sessionId'] ?? '';

    return BattleScreen(
      initialSymbol: symbol,
      initialName: name,
      initialMarketCode: market,
      initialTrainingStartDate: dateStr.isNotEmpty ? DateTime.tryParse(dateStr) : null,
      replaySessionId: sessionIdStr.isNotEmpty ? int.tryParse(sessionIdStr) : null,
    );
  },
)
```

## 4.8 路由与导航

**设计要点**：
1. **战斗页面作为 Tab**：使用 `IndexedStack` 保持状态，不使用独立路由
2. **从首页跳转**：通过 `GoRouter` 携带参数跳转到 `/battle`
3. **复盘模式**：通过 `sessionId` 参数加载历史训练数据
4. **返回导航**：使用 `Navigator.pop()` 返回上一页

**导航场景**：
| 场景 | 导航方式 | 状态保持 |
|------|---------|---------|
| Tab 切换（首页↔实战） | IndexedStack | 完全保持 |
| 首页→实战（选股） | GoRouter | 保持（Provider） |
| 实战→复盘详情 | GoRouter | 保存到DB |
| 实战→训练结束 | 弹窗 | 无需导航 |
| 训练结束→记录页 | Navigator.pop + GoRouter | 状态保存到DB |

```
Provider 层（Notifier）

Service 层
    ↓ 捕获数据访问异常
Repository / DAO 层
    ↓ 捕获基础设施异常
```

```dart
class PerformanceMonitor {
  static final Logger _logger = Logger();

  static void track(String operation, Duration duration) {
    if (duration.inMilliseconds > 500) {
      _logger.w(
        '性能告警: $operation 耗时过长',
        extra: {'duration': '${duration.inMilliseconds}ms'},
      );
    }
  }

  static Future<T> trackAsync<T>(String operation, Future<T> future) async {
    final stopwatch = Stopwatch()..start();
    try {
      return await future;
    } finally {
      stopwatch.stop();
      track(operation, stopwatch.elapsed);
    }
  }
}
```

```dart
// lib/features/battle/utils/symbol_converter.dart

class SymbolConverter {
  // 数据库格式 → 前端显示格式
  static String toDisplay(String dbSymbol, String marketCode) {
    if (dbSymbol.startsWith('SH') || dbSymbol.startsWith('SZ')) {
      return dbSymbol;  // 已经是显示格式
    }
    // 纯数字代码 + 市场代码 → 显示格式
    final prefix = marketCode == 'XSHE' ? 'SZ' : 'SH';
    return '$prefix$dbSymbol';
  }

  // 前端输入 → 数据库查询格式
  static String toDbQuery(String displaySymbol) {
    String normalized = displaySymbol;

    // 移除市场前缀 (SH, SZ)
    if (normalized.startsWith('SH') && normalized.length > 2) {
      normalized = normalized.substring(2);
    } else if (normalized.startsWith('SZ') && normalized.length > 2) {
      normalized = normalized.substring(2);
    }

    // 移除后缀 (.XSHE, .XSHG)
    if (normalized.contains('.')) {
      normalized = normalized.split('.')[0];
    }

    return normalized;
  }

  // 市场代码转换
  static String toExchangeCode(String marketPrefix) {
    switch (marketPrefix) {
      case 'SH':
        return 'XSHG';
      case 'SZ':
        return 'XSHE';
      default:
        return marketPrefix;
    }
  }
}
```

### 15.3 日期时间格式

**现状问题**：
| 层级 | 格式 | 示例 |
|------|------|------|
| 数据库 | DateTime / Text | `2023-03-31` 或 `2023-03-31T00:00:00` |
| 路由参数 | ISO8601 String | `2023-03-31T00:00:00.000` |
| 交易记录 | Text | `2023-03-31` |

```dart
// lib/features/battle/utils/date_converter.dart

class DateConverter {
  // 存储格式：ISO8601 (用于数据库和路由)
  static const String storageFormat = "yyyy-MM-dd'T'HH:mm:ss";

  // 显示格式：中文年月日
  static const String displayFormat = 'yyyy-MM-dd';

  // 短显示格式：月日
  static const String shortDisplayFormat = 'MM-dd';

  // DateTime → 存储格式 (数据库/路由)
  static String toStorage(DateTime date) {
    return date.toIso8601String();
  }

  // 存储格式 → DateTime
  static DateTime fromStorage(String storage) {
    return DateTime.parse(storage);
  }

  // DateTime → 显示格式
  static String toDisplay(DateTime date) {
    return '${date.year}-${date.month.toString().padLeft(2, '0')}-${date.day.toString().padLeft(2, '0')}';
  }

  // 显示格式 → DateTime
  static DateTime fromDisplay(String display) {
    final parts = display.split('-');
    return DateTime(
      int.parse(parts[0]),
      int.parse(parts[1]),
      int.parse(parts[2]),
    );
  }

  // 短显示格式 (月-日)
  static String toShortDisplay(DateTime date) {
    return '${date.month.toString().padLeft(2, '0')}-${date.day.toString().padLeft(2, '0')}';
  }
}
```

```dart
Future<List<KlineDataData>> getKlineData(
  String symbol,
  String period, {
  DateTime? startDate,
  DateTime? endDate,
}) {
  final query = select(klineData)
    ..where((t) => t.symbol.equals(symbol))
    ..where((t) => t.period.equals(period))
    ..orderBy([(t) => OrderingTerm.asc(t.tradeDate)]);

  if (startDate != null) {
    // 使用 DateTime 直接比较，Drift 会正确处理
    query.where((t) => t.tradeDate.isBiggerOrEqualValue(startDate));
  }
  if (endDate != null) {
    query.where((t) => t.tradeDate.isSmallerOrEqualValue(endDate));
  }

  return query.get();
}
```

### 15.6 无合格股票时的处理策略

**核心原则**：**禁止使用模拟数据**，所有数据必须从数据库真实获取

**处理流程**：
```
1. 查询数据库获取有至少210天日线数据的股票列表
       ↓
2. 如果列表为空 → 显示"数据库中暂无合格股票，请先同步数据"
       ↓
3. 如果列表不为空 → 随机选择一只股票
       ↓
4. 如果选中的股票数据不足210天 → 尝试下一只股票（最多尝试3次）
       ↓
5. 如果所有候选股票都数据不足 → 尝试扩大时间范围查找更多股票
       ↓
6. 如果仍然找不到合格股票 → 显示"数据不足，无法进行训练"
```

**代码示例**：
```dart
// lib/features/battle/services/stock_service.dart

class StockService {
  final DatabaseService _db;
  final KlineRepository _repository;

  StockService({DatabaseService? db, KlineRepository? repository})
      : _db = db ?? DatabaseService.instance,
        _repository = repository ?? KlineRepository();

  /// 查找合格股票（至少210天数据）
  Future<CandidateStock?> findQualifiedStock({int minDays = 210}) async {
    // 1. 查询有足够数据的股票
    final candidates = await _db.klineDao.getSymbolsWithMinKlineData(
      minDays: minDays,
    );

    if (candidates.isEmpty) {
      return null;  // 无合格股票
    }

    // 2. 随机选择一只
    final random = Random();
    final shuffled = List.of(candidates)..shuffle(random);

    // 3. 验证候选股票的数据量（最多尝试3只）
    for (int i = 0; i < shuffled.length && i < 3; i++) {
      final candidate = shuffled[i];
      final symbol = candidate['symbol']!;
      final marketCode = candidate['marketCode']!;

      // 验证数据量
      final klineCount = await _db.klineDao.getKlineDataCount(symbol);
      if (klineCount >= minDays) {
        return CandidateStock(
          symbol: symbol,
          marketCode: marketCode,
          dataDays: klineCount,
        );
      }
    }

    // 4. 尝试扩大时间范围
    final allSymbols = await _db.klineDao.getSymbols();
    for (final symbol in allSymbols) {
      final klineCount = await _db.klineDao.getKlineDataCount(symbol.symbol);
      if (klineCount >= minDays) {
        return CandidateStock(
          symbol: symbol.symbol,
          marketCode: symbol.marketCode,
          dataDays: klineCount,
        );
      }
    }

    return null;  // 仍然找不到
  }
}

class CandidateStock {
  final String symbol;
  final String marketCode;
  final int dataDays;

  const CandidateStock({
    required this.symbol,
    required this.marketCode,
    required this.dataDays,
  });
}
```
