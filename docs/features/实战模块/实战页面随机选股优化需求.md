# 实战页面随机选股优化 — 功能需求说明书

## 1. 背景与价值

**为什么做**：当前实战页面初始化时调用随机选股功能，但从数据库获取符合条件的数据时返回 0 条记录，最终只能使用保底股票。原因是 `symbols` 表与 `kline_data` 表的 symbol 格式不统一，导致查询无法匹配到数据。

**做成什么样**：优化实战页面的随机选股逻辑，直接从 `kline_data` 表查询有足够数据的股票（不依赖 symbols 表），确保数据库中有真实 K 线数据的股票能够被正确查询到。

**关联**：
- 关联功能：复盘页面（复盘功能依赖正确加载训练数据）
- 关联问题：K线数据查询返回0条问题

---

## 2. 范围

### 本次做

- 优化实战页面初始化时的随机选股逻辑
- 新增直接从 `kline_data` 表查询股票的方法
- 统一 symbol 参数格式，确保参数与数据库保持一致

### 本次不做

- symbols 表数据修复或同步
- 已有功能（非随机选股路径）的参数格式调整
- K线数据导入逻辑修改

## 4. 需求详述

### 4.1 核心流程

1. 用户进入实战页面（无路由参数）
2. 系统自动从 `kline_data` 表查询有完整数据的股票列表
3. 系统随机选择一只股票
4. 系统加载该股票的 K 线数据并显示

### 4.2 交互规则

- 初始化过程中显示加载状态
- 加载失败时显示友好提示，使用保底股票
- 不影响从首页选股跳转的已有功能

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | symbol 参数格式统一为纯数字格式（如 `600519`），与数据库 `kline_data` 表保持一致 |
| BR-002 | 随机选股只查询日线数据（period='day'） |
| BR-003 | 股票必须至少有 210 天数据（150 天训练 + 60 天预热） |
| BR-004 | 如果数据库没有符合条件的股票，返回空列表，页面显示"暂无数据"提示 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 实战页面随机选股成功加载数据
  - **Given** 用户进入实战页面且数据库中有 K 线数据
  - **When** 系统执行随机选股逻辑
  - **Then** 从 kline_data 表查询到符合条件的股票，并加载其 K 线数据

- [ ] **AC-002**: K 线数据显示正常
  - **Given** 随机选股成功
  - **When** K 线数据加载完成
  - **Then** 页面显示完整的 K 线图表，数据条数 = 历史天数 + 训练天数

- [ ] **AC-003**: 实战页面初始化参数格式统一
  - **Given** 系统执行随机选股
  - **When** 查询 kline_data 表
  - **Then** symbol 参数格式为纯数字（如 `600519`），与数据库存储格式一致

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-004**: 数据库无数据时显示空状态提示
  - **Given** 数据库中没有 >= 210 条数据的股票
  - **When** 系统执行随机选股
  - **Then** 返回空列表，页面显示"暂无可训练股票"提示，不使用硬编码的保底股票

- [ ] **AC-005**: 已有功能不受影响
  - **Given** 用户从首页选股跳转到实战页面
  - **When** 页面初始化
  - **Then** 正常加载传递的股票数据，不受随机选股逻辑影响

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-006**: symbol 参数格式验证
  - **Given** 系统需要查询 kline_data 表
  - **When** 执行 SQL 查询
  - **Then** symbol 参数格式为纯数字（如 `600519`、`000021`），确保能匹配数据库中的记录

- [ ] **AC-007**: 数据量要求验证
  - **Given** 随机选股查询
  - **When** 检查股票数据完整性
  - **Then** 股票必须至少有 210 条日线数据（150 天训练 + 60 天预热）
