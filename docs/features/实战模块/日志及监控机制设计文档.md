# 日志及监控机制设计文档

## 1. 概述

本文档描述了 KlineTrain 项目的日志及监控机制设计，包括日志系统、性能监控、健康监控三个核心模块，旨在提升系统的可观测性和可维护性。

## 2. 日志系统

### 2.1 设计目标

- **分级管理**：支持多种日志级别，便于过滤和筛选
- **分类记录**：按业务模块分类，便于定位问题
- **持久化存储**：日志自动保存到本地文件，便于追溯
- **高性能**：采用缓冲机制，避免频繁IO操作

### 2.2 日志级别

| 级别 | 名称 | 用途 | 颜色标识 |
|------|------|------|----------|
| v | Verbose | 详细调试信息 | 灰色 |
| d | Debug | 调试信息 | 蓝色 |
| i | Info | 一般信息 | 绿色 |
| w | Warning | 警告信息 | 黄色 |
| e | Error | 错误信息 | 红色 |
| wtf | WTF | 严重错误 | 红色加粗 |

### 2.3 日志分类

| 分类 | 标识 | 用途 |
|------|------|------|
| app | APP | 应用级日志 |
| database | DB | 数据库操作日志 |
| network | NET | 网络请求日志 |
| performance | PERF | 性能监控日志 |
| security | SEC | 安全相关日志 |
| business | BUS | 业务逻辑日志 |
| ui | UI | 用户界面日志 |

### 2.4 文件结构

```
lib/shared/utils/logger.dart    # 日志系统核心实现
```

### 2.5 使用示例

```dart
import 'package:kline_train/shared/utils/logger.dart';

// 普通日志
appLogger.i('用户登录成功', category: LogCategory.business);

// 带错误信息的日志
appLogger.e('数据库查询失败', 
    error: e, 
    stackTrace: stackTrace, 
    category: LogCategory.database);

// 调试日志
appLogger.d('请求参数: $params', category: LogCategory.network);
```

### 2.6 日志配置

```dart
AppLogger.initialize(const LogConfig(
  minLevel: LogLevel.debug,      // 最小日志级别
  enableConsole: true,           // 启用控制台输出
  enableFile: true,              // 启用文件存储
  maxFileSizeMb: 10,             // 单文件最大大小(MB)
  maxFilesCount: 5,              // 最大文件数
  enabledCategories: [           // 启用的日志分类
    LogCategory.app,
    LogCategory.database,
    LogCategory.network,
  ],
));
```

### 2.7 日志文件存储

日志文件存储在应用文档目录下：

```
<应用文档目录>/app_logs/app.log
```

## 3. 性能监控系统

### 3.1 设计目标

- **性能指标采集**：记录关键操作的耗时
- **性能分析**：提供统计摘要和趋势分析
- **阈值告警**：对超过阈值的操作发出警告
- **持久化存储**：性能数据定期保存到文件

### 3.2 性能指标类型

| 类型 | 名称 | 用途 |
|------|------|------|
| databaseQuery | Database Query | 数据库查询耗时 |
| networkRequest | Network Request | 网络请求耗时 |
| functionExecution | Function Execution | 函数执行耗时 |
| uiRender | UI Render | UI渲染耗时 |
| memoryUsage | Memory Usage | 内存使用情况 |
| cpuUsage | CPU Usage | CPU使用情况 |

### 3.3 文件结构

```
lib/shared/utils/performance_monitor.dart    # 性能监控核心实现
```

### 3.4 使用示例

```dart
import 'package:kline_train/shared/utils/performance_monitor.dart';

// 方式1: 使用 monitor 装饰器
final result = performanceMonitor.monitor(
  '复杂计算',
  PerformanceMetric.functionExecution,
  () => complexCalculation(),
);

// 方式2: 使用 monitorAsync 装饰器
final data = await performanceMonitor.monitorAsync(
  '数据库查询',
  PerformanceMetric.databaseQuery,
  () => database.query(),
);

// 方式3: 使用性能跟踪器
final tracker = performanceMonitor.track('文件上传', PerformanceMetric.networkRequest);
try {
  await uploadFile();
} finally {
  tracker.stop();
}
```

### 3.5 性能统计

```dart

print(jsonEncode(summary));

// 输出示例
```

## 4. 健康监控系统

### 4.1 设计目标

- **健康检查**：定期检查关键组件状态
- **状态报告**：提供整体健康状态评估
- **历史记录**：保存检查历史，便于趋势分析
- **自动告警**：对异常状态及时发现

### 4.2 健康状态

| 状态 | 名称 | 含义 |
|------|------|------|
| healthy | Healthy | 健康，所有检查通过 |
| degraded | Degraded | 降级，部分检查警告 |
| unhealthy | Unhealthy | 不健康，存在严重问题 |

### 4.3 文件结构

```
lib/shared/utils/health_monitor.dart    # 健康监控核心实现
```

### 4.4 使用示例

```dart
import 'package:kline_train/shared/utils/health_monitor.dart';

// 注册健康检查
healthMonitor.registerCheck('database', () async {
  try {
    await database.ping();
    return HealthCheckResult(
      component: 'database',
      status: HealthStatus.healthy,
      message: '数据库连接正常',
    );
  } catch (e) {
    return HealthCheckResult(
      component: 'database',
      status: HealthStatus.unhealthy,
      message: '数据库连接失败: $e',
    );
  }
});

// 执行所有检查
final results = await healthMonitor.runAllChecks();

// 获取整体状态
final overallStatus = healthMonitor.getOverallStatus();

// 启动定时检查（每5分钟）
healthMonitor.startPeriodicChecks(interval: Duration(minutes: 5));
```

健康检查日志存储在应用文档目录下：



## 6. 最佳实践

### 6.1 日志使用规范

1. **使用合适的日志级别**：
   - `v`：详细调试，生产环境关闭
   - `d`：调试信息，生产环境可关闭
   - `i`：重要业务流程记录
   - `w`：潜在问题警告
   - `e`：错误必须记录

2. **添加日志分类**：
   - 数据库操作使用 `LogCategory.database`
   - 网络请求使用 `LogCategory.network`
   - 业务逻辑使用 `LogCategory.business`

3. **错误日志必须包含堆栈**：
   ```dart
   try {
     // 操作
   } catch (e, stackTrace) {
     appLogger.e('操作失败', error: e, stackTrace: stackTrace);
   }
   ```

### 6.2 性能监控规范

1. **监控关键路径**：
   - 数据库查询
   - 网络请求
   - 复杂计算
   - UI渲染

2. **设置合理阈值**：
   - 数据库查询：>1000ms 警告
   - 网络请求：>5000ms 警告

### 6.3 健康检查规范

1. **定期检查**：
   - 数据库连接
   - 关键服务状态
   - 存储可用性

2. **检查频率**：
   - 关键组件：每1分钟
   - 一般组件：每5分钟

## 8. 注意事项

1. **敏感信息保护**：日志中不得记录密码、Token等敏感信息
2. **日志清理**：定期清理过期日志文件，避免占用过多存储空间
3. **性能影响**：生产环境可适当降低日志级别，减少性能开销
4. **权限管理**：日志文件应设置适当的访问权限，防止未授权访问
