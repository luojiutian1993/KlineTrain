# 训练周期结束处理需求

## 1. 背景与价值

### 1.1 为什么做

当前实战页面存在以下问题：
1. **训练周期结束判断缺失**：150天训练结束后，点击"下一步"按钮仍可继续，且展示后续数据
2. **训练记录保存缺失**：训练结束后没有自动保存训练记录的逻辑
3. **数据使用逻辑混淆**：额外查询的20天缓冲数据被当作训练数据使用，造成用户困惑

**根本原因**：
- `handleNextStep()` 方法中没有检查训练是否已经结束
- 额外查询的缓冲数据被用于训练，造成"150天后还能继续"的假象
- 训练结束对话框和保存逻辑未实现

### 1.2 做成什么样

实现训练周期结束的完整处理流程：
- 150天训练结束后，点击"下一步"应弹出提示"训练已结束！"
- 训练结束后自动保存训练记录到数据库
- 训练结束后，用户可以切换到复盘模式查看结果，但不能继续训练
- 后续数据仅用于数据充足性保障，不展示给用户

### 1.3 关联

- 现有数据加载逻辑（`battle_provider.dart` 的 `_loadKlineData` 方法）
- 现有训练周期配置（`BattleConfig.defaultTrainingDays = 150`）
- 训练记录保存逻辑（`TrainingDao`）
- 复盘模式功能（`loadReplayMode`）

## 3. 数据分层结构

### 3.1 数据分层定义

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 数据充足性保障层（额外20天缓冲）                                              │
│ 这部分数据不展示给用户，仅用于确保数据查询结果满足前端需求                         │
│                                                                            │
│ ┌────────────────────────────────────────────────────────────────────┐    │
│ │ 历史展示层（100天预加载 + 33天指标前置 = 133天）                          │    │
│ │ 训练起始索引 = 133                                                     │    │
│ │ 用于支持用户左滑查看历史数据和指标预热                                    │    │
│ │                                                                            │    │
│ │ ┌──────────────────────────────────────────────────────────────┐    │    │
│ │ │ 训练周期层（150天工作日）                                            │    │    │
│ │ │ 训练范围：索引 133 ~ 282（共150天）                                │    │    │
│ │ │ 用户主要训练区域，可点击"下一步"推进                                  │    │    │
│ │ │ 训练结束后不能继续推进                                              │    │    │
│ │ └──────────────────────────────────────────────────────────────┘    │    │
│ └────────────────────────────────────────────────────────────────────┘    │
│                                                                            │
│ 最终加载范围：约320个日历天（对应283个工作日 + 20天缓冲）                        │
└────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 索引计算公式

| 变量 | 计算方式 | 说明 |
|------|---------|------|
| **训练起点索引** | `historyDays = 133` | 固定值：100（预加载） + 33（指标前置） |
| **训练终点索引** | `trainingEndIndex = initialStartIndex + trainingDays - 1` | 训练起点 + 训练天数 - 1 |
| **训练天数** | `trainingDays = 150` | 固定配置值 |
| **可用数据上限** | `allKlineData.length - 1` | 数据库实际数据长度 - 1 |

### 3.3 数据使用规则

| 数据类型 | 索引范围 | 使用场景 | 是否展示 |
|---------|---------|---------|---------|
| **缓冲数据** | 0 ~ (initialStartIndex - 1) | 数据充足性保障 | ❌ 不展示 |
| **历史展示数据** | initialStartIndex - buffer ~ initialStartIndex | 左滑查看历史 | ✅ 展示 |
| **训练数据** | initialStartIndex ~ trainingEndIndex | 用户点击下一步推进 | ✅ 展示 |
| **后续数据** | (trainingEndIndex + 1) ~ end | 数据充足性保障 | ❌ 不展示 |

## 4. 功能设计

### 4.1 训练周期结束判断

**判断条件**：
```dart
bool isTrainingComplete() {
  return currentDayIndex >= trainingEndIndex;
}

int get trainingEndIndex {
  return initialStartIndex + trainingDays - 1;
}
```

**判断时机**：
- 在 `handleNextStep()` 方法中
- 在 `nextDay()` 方法中
- 在 UI 更新时（如进度显示）

### 4.2 训练结束提示

**提示方式**：弹出对话框

**对话框内容**：
| 元素 | 内容 |
|------|------|
| 标题 | "训练结束" |
| 内容 | "恭喜完成 ${trainingDays} 天训练！" |
| 统计信息 | 收益率、交易次数、胜率等 |
| 按钮1 | "查看复盘" → 切换到复盘模式 |
| 按钮2 | "重新开始" → 重新选择股票开始训练 |

**交互流程**：
```
1. 用户点击"下一步"
2. 系统检查 currentDayIndex >= trainingEndIndex
3. 如果训练结束：
   a. 显示训练结束对话框
   b. 保存训练记录
   c. 不推进 currentDayIndex
4. 如果训练未结束：
   a. 正常推进一天
   b. 更新 UI
```

### 4.3 训练记录保存

**保存时机**：训练结束时（显示对话框时）

**保存内容**：
| 字段 | 来源 |
|------|------|
| symbol | state.currentSymbol |
| marketCode | state.currentMarketCode |
| startDate | state.trainingStartDate |
| endDate | 当前K线日期（trainingEndIndex对应的日期） |
| initialCapital | state.initialBalance |
| finalCapital | state.accountBalance + state.positionValue |
| totalProfit | state.totalProfitLoss |
| tradeCount | state.tradePoints.length |
| winCount | 计算盈利交易次数 |
| winRate | winCount / tradeCount |

**保存方法**：
```dart
Future<int> saveTrainingRecord() async {
  final dbService = DatabaseService.instance;

  final sessionId = await dbService.trainingDao.createSession(
    TrainingSessionsCompanion.insert(
      symbol: state.currentSymbol,
      marketCode: state.currentMarketCode,
      startDate: state.trainingStartDate ?? DateTime.now(),
      endDate: state.allKlineData[state.currentDayIndex].dateTime,
      initialCapital: state.initialBalance,
      finalCapital: state.accountBalance + state.positionValue,
      totalProfit: state.totalProfitLoss,
      tradeCount: state.tradePoints.length,
      winCount: _calculateWinCount(),
      winRate: _calculateWinRate(),
      createdAt: DateTime.now(),
    ),
  );

  // 保存每笔交易
  for (final point in state.tradePoints) {
    await dbService.trainingDao.addTrade(
      TradesCompanion.insert(
        sessionId: sessionId,
        type: point.isBuy ? 'buy' : 'sell',
        price: point.price,
        quantity: point.quantity,
        tradeDate: DateFormat('yyyy-MM-dd').format(point.date),
        createdAt: DateTime.now(),
      ),
    );
  }

  return sessionId;
}
```

### 4.4 训练结束后的行为控制

**"下一步"按钮**：
- 训练结束后：禁用或显示"已完成"
- 复盘模式下：可继续查看历史数据

**其他按钮**：
| 按钮 | 训练结束前 | 训练结束后 |
|------|---------|---------|
| 换股 | ✅ 可用 | ✅ 可用（重新开始） |
| 条件单 | ✅ 可用 | ❌ 禁用 |
| 买入 | ✅ 可用 | ❌ 禁用 |
| 卖出 | ✅ 可用 | ❌ 禁用（显示提示） |
| 下一步 | ✅ 可用 | ❌ 禁用/显示"已完成" |
| 查看复盘 | - | ✅ 可用 |

## 7. 技术实现要点

### 7.1 修改点

1. **BattleState** - 添加 `trainingEndIndex` getter
2. **BattleProvider.handleNextStep()** - 添加训练结束判断
3. **BattleProvider.nextDay()** - 添加边界检查
4. **BattleProvider.saveTrainingRecord()** - 新增训练记录保存方法
5. **BattleScreen** - 更新 UI 状态控制，显示训练结束对话框

### 7.2 核心逻辑修改

```dart
// 1. BattleState 添加训练终点索引
int get trainingEndIndex {
  return initialStartIndex + trainingDays - 1;
}

bool get isTrainingComplete {
  return currentDayIndex >= trainingEndIndex;
}

// 2. handleNextStep() 修改
void handleNextStep() {
  if (state.isReplayMode) {
    // 复盘模式逻辑不变
    return;
  }

  if (state.isTrainingComplete) {
    // 训练已结束，显示对话框并保存记录
    _showTrainingCompleteDialog();
    return;
  }

  if (state.phase == TrainingPhase.opening) {
    setPhase(TrainingPhase.closing);
  } else {
    nextDay();
    setPhase(TrainingPhase.opening);
  }
}

// 3. nextDay() 边界检查
void nextDay() {
  final newIndex = state.currentDayIndex + 1;

  // 检查是否到达训练终点
  if (newIndex > state.trainingEndIndex) {
    return;
  }

  state = state.copyWith(
    currentDayIndex: newIndex,
    phase: TrainingPhase.closing,
    visibleStartIndex: max(0, newIndex - state.visibleKlineCount + 1),
  );
}

// 4. 显示训练结束对话框
void _showTrainingCompleteDialog() async {
  // 保存训练记录
  final sessionId = await saveTrainingRecord();

  // 显示对话框
  showDialog(
    context: context,
    builder: (context) => TrainingCompleteDialog(
      sessionId: sessionId,
      profitRate: state.totalProfitLoss / state.initialBalance * 100,
      tradeCount: state.tradePoints.length,
      onReplay: () {
        Navigator.pop(context);
        ref.read(battleProvider.notifier).loadReplayMode(sessionId);
      },
      onRestart: () {
        Navigator.pop(context);
        ref.read(battleProvider.notifier).initializeRandom();
      },
    ),
  );
}
```

### 7.3 UI 状态控制

```dart
Widget _buildTradeButtons() {
  final isComplete = ref.watch(battleProvider.select((s) => s.isTrainingComplete));

  return Row(
    children: [
      // 换股按钮（训练结束后可重新开始）
      Expanded(child: _buildButton('换股', ...)),

      // 条件单（训练结束后禁用）
      if (!isComplete)
        Expanded(child: _buildButton('条件单', ...)),

      // 买入（训练结束后禁用）
      if (!isComplete)
        Expanded(child: _buildButton('买入', ...)),

      // 卖出（训练结束后禁用）
      if (!isComplete)
        Expanded(child: _buildButton('卖出', ...)),

      // 下一步（训练结束后显示"已完成"）
      Expanded(
        child: _buildButton(
          isComplete ? '已完成' : '下一步',
          isComplete ? () {} : () => ref.read(battleProvider.notifier).handleNextStep(),
          isComplete ? Colors.grey : AppTheme.accent,
          Colors.white,
        ),
      ),
    ],
  );
}
```
