# 选股查询效率优化与防抖功能技术方案（修订版）

## 1. 需求分析

**需求内容：**
1. **提升查询效率**：当前首页选取选股条件时，后台查询很慢，需要提升到用户感觉秒选的效果
2. **添加防抖功能**：多次点击选股条件时，不能每次点击都查询系统

## 2. 问题诊断

### 2.1 问题描述
用户反馈选股查询存在以下问题：
1. 查询效率慢
2. 使用选股条件查找出来0只股票

### 2.2 问题分析
经过分析，发现批量查询SQL存在以下问题：

1. **SQL语法问题**：使用了窗口函数 `ROW_NUMBER() OVER (PARTITION BY symbol ORDER BY trade_date DESC)`
   - SQLite对窗口函数支持有限
   - 可能导致查询结果不正确

2. **LEFT JOIN逻辑问题**：使用LEFT JOIN连接当前日期和N天前的数据
   - 如果past.close为NULL，则return_rate会是NULL
   - 导致查询结果为空

### 2.3 解决方案
采用更可靠的方式：
1. 恢复使用原来的循环查询方式（经过验证可以工作）
2. 保留防抖机制（300ms延迟）
3. 添加详细的日志记录，便于问题诊断

## 3. 技术方案（修订版）

### 方案一：添加防抖机制（已实现）✅

在 Provider 层添加防抖，延迟执行查询，避免快速点击导致的重复查询。

### 方案二：恢复循环查询（已实现）✅

恢复使用原来的循环查询方式：
```dart
for (final symbol in allSymbols) {
  final ret = await calculateReturnNDays(symbol.symbol, date, 30, ...);
  if (ret != null) {
    returns[symbol.symbol] = ret;
  }
}
```

这种方式经过验证可以正常工作。

### 方案三：添加日志记录（已实现）✅

添加详细的日志记录，便于问题诊断：
```dart
appLogger.i('getReturn30dTop50 - 开始计算30日涨幅前50%');
appLogger.w('getReturn30dTop50 - 未找到符合条件的股票');
appLogger.i('getReturn30dTop50 - 找到 ${sorted.length} 支股票，返回前 ${cutoffIndex} 支');
```

## 6. 后续优化建议

### 6.1 数据库索引优化
在 `kline_data` 表上添加复合索引：
```sql
CREATE INDEX idx_kline_symbol_date ON kline_data(symbol, period, trade_date);
```

### 6.2 批量查询优化（待实现）
在保证查询正确性的前提下，使用更简单的SQL实现批量查询：
```sql
-- 使用子查询替代窗口函数
SELECT 
    current.symbol,
    (current.close / past.close - 1) * 100 as return_rate
FROM kline_data current
JOIN kline_data past ON current.symbol = past.symbol
WHERE current.trade_date = ?
  AND past.trade_date = date(?, '-' || ? || ' days')
  AND current.period = 'day'
  AND past.period = 'day'
```
