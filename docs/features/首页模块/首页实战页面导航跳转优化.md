# 首页-实战页面导航跳转优化

## 1. 背景与价值

**为什么做**：当前实战页面存在导航跳转问题：
1. 第一次从首页选股可以正常跳转到实战页面并显示对应股票
2. 第二次从首页选股时，路由参数正确传递但UI仍然停留在首页，无法自动切换到实战页面Tab
3. 复盘功能跳转也存在同样的问题

**做成什么样**：
- 无论第几次点击，都能正确从首页/复盘跳转到实战页面并切换到对应Tab
- 路由参数变化时，强制更新UI当前显示的Tab页
- 用户体验流畅，无感知的技术实现

**关联**：本功能涉及 `MainTabPage`、`BattleScreen`、`HomeScreen`、`RecordsScreen` 等多个文件，属于导航架构优化的关键Bug修复。

## 4. 需求详述

### 4.1 问题分析

#### 问题现象

1. **首次点击**：✅ 正常工作
   - 创建新的 `MainTabPage` 实例
   - `initState()` 中 `_currentIndex` 被设为 `1`
   - `IndexedStack` 显示 BattleScreen

2. **第二次点击**：❌ UI不切换
   - `MainTabPage` 实例已存在
   - GoRouter 复用现有实例，只更新 `battleParams`
   - `initState()` 不会再调用
   - `_currentIndex` 保持原值（可能是0）
   - `IndexedStack` 仍然显示 HomeScreen

#### 问题根因

```
GoRouter 路由更新流程：
1. 用户点击"确认"按钮
2. context.go('/main?tab=1&symbol=...')
3. GoRouter 检测到 /main 路由已存在
4. GoRouter 复用现有 MainTabPage 实例
5. MainTabPage 的 widget 属性被更新（battleParams 变化）
6. 但 StatefulWidget 的 initState() 不会重新调用
7. _currentIndex 保持原值
8. IndexedStack 仍然显示旧页面
```

### 4.2 解决方案

#### 方案：添加 didUpdateWidget 生命周期处理

在 `MainTabPage` 中添加 `didUpdateWidget` 方法，监听 `battleParams` 变化并强制更新 `_currentIndex`。

```dart
@override
void didUpdateWidget(MainTabPage oldWidget) {
  super.didUpdateWidget(oldWidget);
  if (widget.battleParams != null) {
    final shouldSwitchToBattle =
        widget.battleParams?.symbol != oldWidget.battleParams?.symbol ||
            widget.battleParams?.trainingStartDate != oldWidget.battleParams?.trainingStartDate;
    if (shouldSwitchToBattle && _currentIndex != 1) {
      setState(() {
        _currentIndex = 1;
      });
    }
  }
}
```

## 5. 验收标准

### 5.1 正常流程

- [ ] **AC-001**: 首次首页选股跳转
  - **Given** 用户从首页选择股票后点击"确认"
  - **When** 路由跳转到 `/main?tab=1&...`
  - **Then** 页面切换到实战Tab，显示对应股票数据

- [ ] **AC-002**: 第二次首页选股跳转
  - **Given** 用户已完成首次跳转，正在实战页面
  - **When** 用户返回首页，选择另一只股票后点击"确认"
  - **Then** 页面切换到实战Tab，显示新选择的股票数据

- [ ] **AC-003**: 连续多次选股跳转
  - **Given** 用户多次在首页选择不同股票
  - **When** 每次点击"确认"后
  - **Then** 每次都能正确跳转到实战Tab

- [ ] **AC-004**: 复盘跳转
  - **Given** 用户在记录页面
  - **When** 点击某条记录的"复盘"按钮
  - **Then** 页面切换到实战Tab，显示该训练记录数据

### 5.2 边界与异常

- [ ] **AC-005**: 相同股票重复选择
  - **Given** 用户选择同一只股票多次
  - **When** 每次点击"确认"
  - **Then** 每次都能正常跳转（允许重复）

- [ ] **AC-006**: 无效路由参数
  - **Given** 路由参数缺少必要字段
  - **When** 进入实战页面
  - **Then** 执行随机选股初始化，不崩溃

### 6.2 关键代码

```dart
// lib/features/main/main_tab_page.dart

class _MainTabPageState extends ConsumerState<MainTabPage> {
  late int _currentIndex;
  bool _initialized = false;

  @override
  void initState() {
    super.initState();
    _currentIndex = widget.initialIndex;
    // 首次进入时的处理
    if (widget.battleParams != null && _currentIndex != 1) {
      _currentIndex = 1;
    }
  }

  @override
  void didUpdateWidget(MainTabPage oldWidget) {
    super.didUpdateWidget(oldWidget);
    // 监听 battleParams 变化
    if (widget.battleParams != null) {
      final shouldSwitchToBattle =
          widget.battleParams?.symbol != oldWidget.battleParams?.symbol ||
              widget.battleParams?.trainingStartDate != oldWidget.battleParams?.trainingStartDate;
      if (shouldSwitchToBattle && _currentIndex != 1) {
        setState(() {
          _currentIndex = 1;
        });
      }
    }
  }

  void _onItemTapped(int index) {
    setState(() {
      _currentIndex = index;
    });
  }
}
```
