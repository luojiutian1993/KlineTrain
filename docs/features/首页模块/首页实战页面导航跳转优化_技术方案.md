# 首页-实战页面导航跳转优化 - 技术方案

## 1. 技术背景

### 1.1 问题描述

当前应用使用 GoRouter 进行路由管理，`MainTabPage` 使用 `IndexedStack` 管理四个 Tab 页面。当从首页或记录页面跳转到实战页面时，存在以下问题：

- **首次跳转**：正常工作，`initState()` 中 `_currentIndex` 被设为 `1`
- **后续跳转**：GoRouter 复用现有 `MainTabPage` 实例，`initState()` 不会重新调用，导致 `_currentIndex` 保持原值，UI 无法切换

### 1.2 技术分析

#### GoRouter 路由更新机制

```
用户点击"确认" → context.go('/main?tab=1&symbol=...')
                                    ↓
        ┌─────────────────────────────┴─────────────────────────────┐
        ↓                                                     ↓
   首次进入                                              已有 /main 路由
   (创建新实例)                                          (复用现有实例)
        ↓                                                     ↓
   initState() 正常执行                                initState() 不执行
        ↓                                                     ↓
   _currentIndex = 1                                   _currentIndex 保持原值
        ↓                                                     ↓
   IndexedStack 显示 BattleScreen                       IndexedStack 显示旧页面
```

#### IndexedStack 工作原理

`IndexedStack` 是 `Stack` 的子类，用于显示单个子组件同时保持所有子组件的状态：
- `index` 属性决定显示哪个子组件
- 修改 `index` 会立即切换显示的子组件
- 隐藏的子组件状态被保留，但不会重建

#### 生命周期分析

| 生命周期 | 触发时机 | 是否触发 |
|---------|---------|---------|
| `constructor` | 创建实例 | ✅ 首次 |
| `initState()` | 插入 Widget 树 | ✅ 首次 |
| `didUpdateWidget()` | 父组件重建，子组件 `widget` 更新 | ✅ 每次更新 |
| `build()` | 渲染 | ✅ 每次状态变化 |

## 2. 技术方案

### 2.1 方案选择

#### 方案A：强制创建新实例（不推荐）

```dart
context.go(
  '/main?tab=1&...',
  extra: {'forceNew': true},  // 强制创建新实例
);
```

**缺点**：
- 需要修改 GoRouter 配置
- 无法区分是"返回"还是"新跳转"
- 状态无法保持（返回时）

#### 方案B：监听路由变化（不推荐）

在 `MainTabPage` 中监听路由变化，手动解析参数并更新状态。

**缺点**：
- 需要访问 `GoRouterState`，增加耦合
- 路由参数解析逻辑重复

#### 方案C：didUpdateWidget 生命周期处理 ✅（推荐）

在 `MainTabPage` 中添加 `didUpdateWidget` 方法，监听 `battleParams` 变化并强制更新 `_currentIndex`。

**优点**：
- 利用 Flutter 生命周期处理子组件更新
- 代码简洁，逻辑清晰
- 无需修改 GoRouter 配置
- 保持状态同时解决导航问题

### 2.2 详细设计

#### 2.2.1 核心逻辑

```dart
@override
void didUpdateWidget(MainTabPage oldWidget) {
  super.didUpdateWidget(oldWidget);
  
  // 检查 battleParams 是否存在
  if (widget.battleParams != null) {
    // 检测参数是否变化（symbol 或 trainingStartDate）
    final shouldSwitchToBattle =
        widget.battleParams?.symbol != oldWidget.battleParams?.symbol ||
        widget.battleParams?.trainingStartDate != oldWidget.battleParams?.trainingStartDate;
    
    // 如果参数变化且当前不是实战Tab，强制切换
    if (shouldSwitchToBattle && _currentIndex != 1) {
      setState(() {
        _currentIndex = 1;
      });
    }
  }
}
```

#### 2.2.2 触发条件

| 条件 | 说明 |
|------|------|
| `widget.battleParams != null` | 路由参数存在（有跳转需求） |
| `battleParams.symbol 变化` | 用户选择了新股票 |
| `battleParams.trainingStartDate 变化` | 用户选择了新训练日期 |
| `_currentIndex != 1` | 当前不是实战Tab（需要切换） |

#### 2.2.3 State 变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `_currentIndex` | `int` | 当前 Tab 索引（0=首页, 1=实战, 2=记录, 3=我的） |
| `_initialized` | `bool` | 是否已完成初始化（用于 `_initAuth` 防重） |

## 3. 实现细节

### 3.1 文件修改

**文件**：`lib/features/main/main_tab_page.dart`

**修改内容**：
1. 在 `initState()` 中添加 battleParams 存在性检查和 index 强制设置
2. 添加 `didUpdateWidget` 方法处理后续更新

### 3.2 代码示例

```dart
class _MainTabPageState extends ConsumerState<MainTabPage> {
  late int _currentIndex;
  bool _initialized = false;

  @override
  void initState() {
    super.initState();
    _currentIndex = widget.initialIndex;
    print('🟡 [MainTabPage] initState, initialIndex=$_currentIndex, battleParams=${widget.battleParams}');
    
    // 首次进入时：如果 battleParams 存在但 index 不是 1，强制切换
    if (widget.battleParams != null && _currentIndex != 1) {
      print('🟡 [MainTabPage] battleParams存在但index不是1，强制切换到1');
      _currentIndex = 1;
    }
    print('🟡 [MainTabPage] 最终_currentIndex=$_currentIndex');
    
    WidgetsBinding.instance.addPostFrameCallback((_) {
      _initAuth();
    });
  }

  @override
  void didUpdateWidget(MainTabPage oldWidget) {
    super.didUpdateWidget(oldWidget);
    print('🟡 [MainTabPage] didUpdateWidget');
    print('🟡   - oldWidget.battleParams: ${oldWidget.battleParams?.symbol}');
    print('🟡   - widget.battleParams: ${widget.battleParams?.symbol}');
    
    if (widget.battleParams != null) {
      // 检测参数是否变化
      final shouldSwitchToBattle =
          widget.battleParams?.symbol != oldWidget.battleParams?.symbol ||
          widget.battleParams?.trainingStartDate != oldWidget.battleParams?.trainingStartDate;
      
      if (shouldSwitchToBattle && _currentIndex != 1) {
        print('🟡 [MainTabPage] battleParams变化且当前不是1，强制切换到1');
        setState(() {
          _currentIndex = 1;
        });
      }
    }
  }

  void _onItemTapped(int index) {
    setState(() {
      _currentIndex = index;
    });
  }
  
  // ... build 方法
}
```
