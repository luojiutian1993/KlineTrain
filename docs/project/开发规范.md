## 4. Dart语言规范

### 4.1 类型系统规范

**✅ 必须使用具体类型**：

```dart
// 正确
final String name = '张三';
final List<KlineModel> items = [];

// 错误
final name = '张三';  // ❌ 推断类型
final items = [];     // ❌ 推断类型
```

### 4.2 禁止的写法

| 禁止 | 原因 | 替代方案 |
|------|------|---------|
| **`dynamic`** | 失去类型安全 | 使用具体类型或泛型 |
| **`var`** | 降低可读性 | 使用明确类型声明 |
| **`Object?`** | 过于宽泛 | 使用具体类型 |
| **非空断言 `!`** | 可能崩溃 | 使用空安全运算符 |
| **类型转换 `as`** | 可能崩溃 | 使用 `is` 检查 |

**❌ 错误示例**：

```dart
// 禁止1：使用dynamic
dynamic data = getData();
(data as Map)['name'];  // ❌

// 禁止2：使用var推断
var data = getData();  // ❌

// 禁止3：非空断言
String! name = getName();  // ❌

// 禁止4：强制类型转换
(user as User).name;  // ❌
```

**✅ 正确示例**：

```dart
// 正确1：使用具体类型
final Map<String, dynamic> data = getData();

// 正确2：使用泛型
final List<KlineModel> items = getItems();

// 正确3：空安全运算符
final String? name = getName();
if (name != null) {
  print(name.length);
}

// 正确4：类型检查
if (user is User) {
  print(user.name);
}
```

### 4.3 不可变对象规范

**✅ 必须使用不可变State**：

```dart
// models/battle_state.dart
class BattleState {
  final String currentSymbol;
  final List<KlineModel> klines;
  final bool isLoading;

  const BattleState({
    this.currentSymbol = '',
    this.klines = const [],
    this.isLoading = false,
  });

  // ✅ 必须提供copyWith方法
  BattleState copyWith({
    String? currentSymbol,
    List<KlineModel>? klines,
    bool? isLoading,
  }) {
    return BattleState(
      currentSymbol: currentSymbol ?? this.currentSymbol,
      klines: klines ?? this.klines,
      isLoading: isLoading ?? this.isLoading,
    );
  }
}
```

**❌ 禁止直接修改状态**：

```dart
// 错误：直接修改
state.items.add(newItem);
state.currentIndex = newIndex;

// 正确：使用copyWith
state = state.copyWith(
  items: [...state.items, newItem],
  currentIndex: newIndex,
);
```

## 5. Flutter框架规范

### 5.1 Widget设计原则

**核心原则**：
1. **单一职责** - 每个Widget只负责一个功能
2. **最小可见** - 只暴露必要的公共API
3. **无状态优先** - 优先使用StatelessWidget
4. **const构造** - 使用const构造函数

### 5.2 Widget组件模板

```dart
// widgets/[component]_widget.dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';

class [Component]Widget extends ConsumerWidget {
  const [Component]Widget({
    super.key,
    // 必要参数
    required this.data,
    // 可选参数
    this.onAction,
  });

  final DataType data;
  final VoidCallback? onAction;

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    // ✅ 通过ref.watch读取Provider
    // final state = ref.watch(battleProvider);

    // ✅ 通过ref.read调用Provider方法
    // final notifier = ref.read(battleProvider.notifier);

    return Container(
      child: Column(
        children: [
          _buildHeader(),
          _buildContent(),
          _buildActions(),
        ],
      ),
    );
  }

  Widget _buildHeader() {
    return Text(data.title);
  }

  Widget _buildContent() {
    return Text(data.content);
  }

  Widget _buildActions() {
    return ElevatedButton(
      onPressed: onAction,
      child: const Text('Action'),
    );
  }
}
```

### 5.3 Screen文件规范

**职责限制**：
- Screen只负责组合Widget和路由参数解析
- 禁止在Screen中直接实现业务逻辑
- 状态管理必须通过Provider

**文件行数限制**：
- ✅ Screen文件 ≤ 400行
- ✅ Provider文件 ≤ 300行
- ✅ State文件 ≤ 200行
- ✅ Service文件 ≤ 300行
- ✅ Widget文件 ≤ 150行

### 5.4 溢出处理规范

| 场景 | 方案 | 代码 |
|------|------|------|
| **文字过长** | Flexible + ellipsis | `Flexible(child: Text(name, overflow: TextOverflow.ellipsis))` |
| **价格数字** | FittedBox缩放 | `FittedBox(fit: BoxFit.scaleDown, child: Text(price))` |
| **按钮文字** | Flexible + FittedBox | `Flexible(child: FittedBox(fit: BoxFit.scaleDown, child: Text(text))))` |
| **列表项** | Expanded | `Expanded(child: Text(content))` |

### 5.5 加载状态规范

| 状态 | UI展示 | 说明 |
|------|--------|------|
| **初始加载** | 全屏Loading + 骨架屏 | 禁止操作 |
| **局部加载** | 按钮Loading + 禁用 | 允许其他操作 |
| **刷新加载** | 下拉刷新 + 指示器 | 等待完成 |
| **错误状态** | 错误提示 + 重试按钮 | 可重试 |

## 8. 代码注释规范

### 8.1 必须写注释的场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **公共API** | 方法、类的用途 | `/// 加载K线数据的公开方法` |
| **业务逻辑** | 复杂算法、计算 | `// 计算盈亏率：(现价-成本)/成本 * 100` |
| **TODO** | 待完成功能 | `// TODO(2026-05-30): 添加单元测试` |
| **FIXME** | 已知问题 | `// FIXME: 极端情况下可能出现崩溃` |
| **为什么** | 非显而易见的设计决策 | `// 使用Random而非UUID，因为需要可复现` |

### 8.2 禁止写注释的场景

| 场景 | 原因 | 替代 |
|------|------|------|
| **显而易见的代码** | 浪费维护成本 | 删除注释 |
| **重复代码说明** | 增加噪音 | 抽取为方法 |
| **无意义的TODO** | 污染代码 | 删除或实现 |

### 8.3 注释风格示例

**✅ 正确注释**：

```dart
/// 初始化战斗状态
///
/// [symbol] 股票代码
/// [startDate] 训练开始日期（可选，默认随机日期）
///
/// 返回初始化后的BattleState
Future<BattleState> initializeWithSymbol({
  required String symbol,
  DateTime? startDate,
}) async { ... }

// 计算盈亏率
// 公式：(现价 - 成本价) / 成本价 * 100
double calculateProfitRate(double currentPrice, double costPrice) {
  return ((currentPrice - costPrice) / costPrice) * 100;
}

// TODO(2026-05-30): 添加单元测试验证盈亏计算
double calculateProfitLoss(double currentPrice, double costPrice, int quantity) {
  return (currentPrice - costPrice) * quantity;
}
```

**❌ 错误注释**：

```dart
// 这是一个方法
Future<void> doSomething() { ... }  // ❌ 废话

// 设置name
void setName(String name) {
  this.name = name;  // ❌ 重复代码
}

// TODO: 实现这个功能
void todoFunction() { ... }  // ❌ 无意义的TODO
```

## 9. 错误处理规范

### 9.1 异常分类

| 异常类型 | 父类 | 使用场景 |
|---------|------|---------|
| **ValidationException** | Exception | 参数校验失败 |
| **NetworkException** | Exception | 网络请求失败 |
| **ServerException** | Exception | 服务器返回错误 |
| **DataException** | Exception | 数据解析错误 |
| **\*FeatureException** | Exception | 业务规则违反 |

### 9.2 异常定义模板

```dart
// exceptions/[feature]_exception.dart

sealed class [Feature]Exception implements Exception {
  final String userMessage;
  final String code;
  final dynamic originalError;

  const [Feature]Exception({
    required this.userMessage,
    required this.code,
    this.originalError,
  });

  @override
  String toString() => '[$runtimeType] $code - $userMessage';
}

class DataLoadException extends [Feature]Exception {
  const DataLoadException({super.originalError})
      : super(
          userMessage: '数据加载失败，请检查网络后重试',
          code: 'DATA_LOAD_ERROR',
        );
}

class InsufficientBalanceException extends [Feature]Exception {
  const InsufficientBalanceException()
      : super(
          userMessage: '可用余额不足',
          code: 'INSUFFICIENT_BALANCE',
        );
}
```

### 9.3 分层错误处理

```
UI层：展示错误
  ↓
Provider层：转换错误为用户消息
  ↓
Service层：捕获并记录错误
  ↓
DAO层：捕获基础设施错误
```

### 9.4 错误展示规范

| 错误类型 | 展示方式 | 持续时间 | 用户操作 |
|---------|---------|---------|---------|
| **网络错误** | Toast提示 | 3秒 | 自动消失 |
| **表单验证** | 输入框下方红色文字 | 直到修正 | 实时显示 |
| **操作失败** | Dialog弹窗 | 确认关闭 | 用户确认 |
| **严重错误** | 全屏错误页 | 直到重试 | 可重试 |

### 14.2 禁止的依赖方向

| 违规 | 原因 | 解决方案 |
|------|------|---------|
| `screens` → `services` | 违反分层 | 通过Provider中转 |
| `providers` → `screens` | 循环依赖 | 重构职责 |
| `services` → `providers` | 下层依赖上层 | 抽取公共逻辑 |
| `widgets` → `screens` | 跨层依赖 | 通过参数传递 |

### 15.3 并发安全

```dart
// Provider状态并发保护
Future<void> nextDay() async {
  if (state.isLoading) return;  // 防止重复触发

  state = state.copyWith(isLoading: true);
  try {
    final newState = _calculateNextDay(state);
    state = newState;
  } finally {
    state = state.copyWith(isLoading: false);
  }
}
```
