# K线训练营 - 需求总览文档

## 1. 项目概述

### 1.1 项目愿景
> 成为国内领先的K线模拟训练平台，帮助初级交易员通过历史真实K线训练提升盘感，降低真实交易的学习成本和风险。

### 1.2 核心价值
- **降低学习门槛**：无需真金白银即可进行真实市场环境的模拟训练
- **提升交易技能**：通过反复练习K线形态识别，提高对短期走势的判断能力
- **建立交易纪律**：通过条件单（止盈止损、网格交易）训练良好的风控习惯
- **数据驱动改进**：基于训练记录统计，清晰了解自身交易风格和短板
- **沉浸式体验**：平滑的K线走势和精准的买卖点位标记，提升训练真实感

### 1.3 目标用户
- **[股票/期货新手]**：刚进入市场，缺乏实战经验，希望先模拟练习再入场
- **[有亏损经历的初级交易员]**：有一定交易经验但胜率不高，希望提升盘感
- **[自我驱动的学习者]**：希望通过系统化训练形成自己的交易体系

## 3. 新增需求详情

### 3.1 首页确认按钮 [REQ-HOME-007]
**功能描述**：在选股条件区域底部添加确认按钮，选择好条件后点击确认跳转到实战页面

**交互流程**：
```
用户选择时间范围 → 用户选择选股条件 → 点击确认按钮 → 跳转到实战页面
```

**验收标准**：
- [ ] 确认按钮在选股条件卡片底部显示
- [ ] 点击确认按钮跳转到实战页面
- [ ] 未选择条件时按钮正常可用

### 3.2 训练周期显示 [REQ-BATTLE-004]
**功能描述**：下一步按钮前显示选股条件中选择的时间周期对应的时长

**交互流程**：
```
用户设置训练周期(60天) → 显示"训练周期: 60天" → 下一步按钮推进K线
```

**验收标准**：
- [ ] 下一步按钮上方显示训练周期时长
- [ ] 训练周期默认值为60天
- [ ] 周期显示样式与整体设计一致

### 3.3 买入/卖出弹窗 [REQ-BATTLE-015] [REQ-BATTLE-016]
**功能描述**：点击买入/卖出按钮弹出独立页面，支持设置买入价格、买入股数，提供1/3仓、1/2仓、全仓等默认选项

**交互流程**：
```
点击买入按钮 → 弹出买入页面 → 设置价格/数量 → 选择仓位快捷选项 → 点击确认/取消
```

**页面元素**：
| 元素 | 描述 | 默认值 |
|------|------|--------|
| 价格输入框 | 支持手动输入价格 | 当前价 |
| 数量输入框 | 支持手动输入数量 | 可买入最大数量 |
| 1/3仓按钮 | 按1/3仓位计算数量 | - |
| 1/2仓按钮 | 按1/2仓位计算数量 | - |
| 全仓按钮 | 按全部可用资金计算数量 | - |
| 金额预览 | 实时显示买入/卖出金额 | - |
| 预计盈亏(卖出) | 实时显示卖出后盈亏 | - |
| 确认按钮 | 执行买入/卖出操作 | - |
| 取消按钮 | 关闭弹窗不执行操作 | - |

**验收标准**：
- [ ] 买入页面价格默认显示当前K线收盘价
- [ ] 买入页面数量默认显示可买入最大数量（100股整数倍）
- [ ] 1/3仓/1/2仓/全仓按钮正确计算数量
- [ ] 买入金额实时更新
- [ ] 卖出页面显示预计盈亏
- [ ] 确认按钮执行买入/卖出操作
- [ ] 取消按钮关闭弹窗

### 3.4 买卖点位标记 [REQ-BATTLE-017]
**功能描述**：在K线图上精准标记用户的买入/卖出位置

**标记样式**：
| 类型 | 颜色 | 方向 | 位置 |
|------|------|------|------|
| 买入标记 | 红色 | 三角形向上 | K线图对应价格位置 |
| 卖出标记 | 绿色 | 三角形向下 | K线图对应价格位置 |

**验收标准**：
- [ ] 买入时在K线图对应位置显示红色向上三角形
- [ ] 卖出时在K线图对应位置显示绿色向下三角形
- [ ] 标记旁显示交易数量标签
- [ ] 多个买卖点都正确显示

### 3.5 平滑均线 [REQ-BATTLE-007]
**功能描述**：使用贝塞尔曲线优化均线渲染，视觉更流畅

**技术实现**：
- 使用 `Path.quadraticBezierTo` 绘制平滑曲线
- isAntiAlias = true 优化渲染质量
- 均线颜色：MA5黄色、MA10紫色、MA20橙色、MA30蓝色

**验收标准**：
- [ ] 均线曲线平滑无锯齿
- [ ] 各周期均线颜色正确
- [ ] 均线数值在图例中正确显示

### 3.6 K线数据从数据库读取 [REQ-BATTLE-020]
**功能描述**：从本地SQLite数据库读取历史K线真实数据

**数据流程**：
```
应用启动 → 检查数据库是否有数据 → 有 → 直接读取
                                         ↓ 无
                              从API获取数据 → 存入数据库 → 返回数据
```

**验收标准**：
- [ ] 优先从本地数据库读取K线数据
- [ ] 数据库无数据时从API获取
- [ ] API数据自动缓存到本地数据库
- [ ] 无网络时使用模拟数据兜底
