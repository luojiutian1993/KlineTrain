# 项目开发规则

## 文档更新规则

### 核心规则

当完成功能开发并更新文档时，**必须**检查 `specs/` 目录下的相关文档是否需要同步更新。

### 规则说明

1. **触发条件**：每次完成功能开发并更新文档后
2. **检查范围**：确认是否需要更新 `specs/` 目录下的以下文档：
   - `功能模块详细需求.md` - 功能需求变化时
   - `技术栈.md` - 使用新技术时
   - `数据库设计文档.md` - 数据库表结构变更时
   - `API文档.md` - 接口变更时
   - `项目结构.md` - 目录结构变化时
   - `开发规范.md` - 开发规范变化时

3. **更新流程**：
   - 完成功能开发
   - 更新 `docs/` 目录下的开发记录文档
   - **检查并更新** `specs/` 目录下的相关规格文档（必须）
   - 在开发记录文档中记录specs文档的更新情况

### 文档对应关系

| 功能变更 | 需要检查的specs文档 |
|----------|---------------------|
| 新增功能模块 | `specs/功能模块详细需求.md`、`specs/项目结构.md` |
| 数据库表变更 | `specs/数据库设计文档.md` |
| 新增API接口 | `specs/API文档.md` |
| 技术栈变化 | `specs/技术栈.md` |
| 目录结构变化 | `specs/项目结构.md` |
| 开发规范更新 | `specs/开发规范.md` |

## AI辅助开发规则

### 项目角色定义

在开发过程中，AI扮演以下角色：
- **项目经理**：需求分析、进度跟踪
- **架构师**：技术选型、代码审查
- **开发者**：功能实现、文档编写

### AI职责

1. **需求分析**：理解用户需求，拆解为具体任务
2. **技术设计**：遵循架构规范，设计合理方案
3. **代码实现**：编写高质量代码，确保可维护性
4. **文档管理**：完成功能后更新相关文档
5. **规则遵守**：遵守本规则文档中的所有规则

### 文档自动化规则

**重要**：AI在完成功能开发后，**必须**自动更新以下文档：

#### 1. 需求汇总文档 (`docs/需求汇总.md`)
- 添加新的需求ID和描述
- 更新已完成需求列表
- 记录版本号

#### 2. 开发记录文档 (`docs/开发记录/`)
- 在 `docs/开发记录/` 目录下创建开发记录
- 文件命名格式：`功能名开发记录.md`
- 记录：需求背景、实现方案、代码变更、测试验证

#### 3. specs目录文档（按需更新）
| 变更类型 | 需要更新的文档 |
|----------|---------------|
| 新增Provider/State | `specs/项目结构.md` |
| 新增数据库表 | `specs/数据库设计文档.md` |
| 新增API接口 | `specs/API文档.md` |
| 新增前端组件 | `specs/项目结构.md`、`specs/功能模块详细需求.md` |
| 新增脚本工具 | `specs/项目结构.md` |

#### 4. 自动更新触发条件
- 完成一个独立功能模块的开发后
- 修改数据库表结构后
- 新增API接口后
- 新增脚本工具后

### 示例

**场景**：用户要求新增"科创板选择"功能

**AI自动执行流程**：
1. 分析需求，设计实现方案
2. 实现代码
3. **自动更新文档**：
   - 更新 `docs/需求汇总.md` - 添加 REQ-013
   - 创建 `docs/开发记录/科创板选择功能开发记录.md`
   - 检查并更新 `specs/项目结构.md`（如有新增文件）
   - 检查并更新 `specs/功能模块详细需求.md`（如有必要）
