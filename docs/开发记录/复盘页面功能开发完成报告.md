## 1. 功能概述

复盘功能允许用户查看已完成训练的历史记录，以可视化的方式回顾整个训练过程，包括 K 线走势和买卖点标记。

**核心实现**: 复用实战页面（BattleScreen）作为复盘界面，锁定在训练结束状态，用户只能浏览不能操作。

## 4. 核心实现细节

### 4.1 路由参数传递

```
RecordsScreen._handleReplay()
  → context.go('/main?tab=1&mode=replay&sessionId=${session.id}&...')
  → AppRoutes /main builder
  → MainTabPage(initialIndex: 1, battleParams: ...)
  → BattleScreen(isReplayMode: true, replaySessionId: sessionId)
```

### 4.2 复盘模式初始化 (_loadReplayModeFromWidget)

1. 从数据库加载 `TrainingSession` 记录
2. 设置账户状态为训练结束时
3. 计算 `_currentDayIndex` 指向最后一天
4. 设置 `_trainingPhase = TrainingPhase.closing`
5. 加载交易记录，构建买卖点列表

### 4.3 UI状态控制

| 操作 | 复盘模式行为 |
|------|-------------|
| 下一步按钮 | 触发 `_showReplayEndDialog()` 弹窗提示 |
| 买入按钮 | 触发 `_showReplayDisabledDialog('买入')` 弹窗，按钮变灰显示"已结束" |
| 卖出按钮 | 触发 `_showReplayDisabledDialog('卖出')` 弹窗，按钮变灰显示"已结束" |
| 进度显示 | 固定显示 `$_trainingDays/$_trainingDays` |

### 4.4 买卖点构建 (_buildTradePoints)

1. 遍历交易记录
2. 根据交易日期匹配 K 线索引
3. 构建 `TradePoint` 列表（包含位置、价格、类型、标签）
4. 将买卖点传给 K 线图组件渲染

## 5. 测试建议

### 5.1 环境准备

1. **清除模拟器内容**: iOS Simulator → Device → Erase All Content and Settings
2. **重新运行应用**: 确保数据库正确初始化
3. **创建训练记录**: 从首页选择股票完成一次训练

### 5.2 测试步骤

1. **跳转测试**: 记录页面 → 点击"复盘" → 验证底部导航栏显示
2. **K线显示**: 验证 K 线图显示到训练最后一天
3. **买卖点标记**: 验证红色买入/绿色卖出三角形标记
4. **进度显示**: 验证显示 `150/150`
5. **按钮禁用**: 点击"下一步/买入/卖出" → 验证弹窗提示
6. **浏览功能**: 测试"后退/前进"按钮和时间滑块
