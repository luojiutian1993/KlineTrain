## 开发概述
根据对话需求，对K线训练营APP的实战页面进行增强，主要包括：
- K线平滑渲染优化
- 买卖点位标记

## 一、需求分析

### 1.1 原始需求
```
实战页面的K线展示，需要更适配机器，k线走势需要更加顺滑，同时需要展示买入卖出的点位，在换股、条件单、买入、卖出的模块，新增"下一步"按钮，通过点击下一步，实现K线每天的变化。k线的变化需要从数据库读取真实数据，展示到页面中
```

### 1.2 需求细化
根据对话确认，进一步明确：
- **下一步按钮**：点击后K线推进一天，从数据库加载真实数据
- **换股**：弹出选股页面，切换标的
- **条件单**：弹出条件单设置页面
- **买入**：弹出买入页面，支持价格/数量/仓位选项
- **卖出**：弹出卖出页面，支持价格/数量/仓位选项

### 1.3 补充需求
```原文
1. 在首页选股条件新增确认按钮，选择好条件后，点击确认跳转到实战页面
2. 实战页面的图片对应的模块需要删除
3. 实战页面的下一步按钮之前展示选股条件中选择的时间周期对应的时长（例如20天、60天）
4. 点击买入按钮，弹出新的页面，支持选择买入价格、买入股数，同时有1/2仓、1/3仓、全仓等默认选项，点击确认后完成买入，点击取消按钮返回到原页面
5. 点击卖出按钮，功能与买入类似
```

```dart
void _drawSmoothLine(Canvas canvas, Size size, List<double> values, Color color) {
```

```dart
class TradePoint {
```

#### 2.2.4 数据库优先读取
```dart
Future<List<KlineModel>> fetchKlineData(...) async {
  // 1. 优先从数据库读取
  final dbData = await fetchKlineDataFromDb(...);
  if (dbData.isNotEmpty) return dbData;

  // 2. 数据库没有从API获取
  final apiData = await _api.fetchKlineData(...);
  if (apiData.isNotEmpty) {
    await _saveToDatabase(apiData, timeframe);
    return apiData;
  }

  // 3. 兜底使用模拟数据
  return _generateMockData(symbol, limit);
}
```

### 6.1 指标区域重构 [2026-05-16]
**问题**：指标下拉框和图表分开显示，切换指标时图表叠加而不是更新

**原因分析**：
1. KlineChart组件内部已经包含了成交量和MACD图表渲染
2. 实战页面又通过 `_buildIndicatorCharts()` 再次渲染指标图表
3. 导致页面显示4个图表（K线图内部的2个 + 实战页面控制的2个）

**修复方案**：
1. 从 KlineChart 组件中移除成交量和MACD图表，只保留K线价格图表
2. 重构 `_buildIndicatorSection` 方法，将下拉框和图表放在同一个区域内

**修改文件**：

- `lib/features/battle/battle_screen.dart`

### 5.4 修复图表 `horizontalInterval` 为 0 的错误
**时间**: 2026-05-16

**问题原因**:
- 当所有成交量数据或MACD数据都是 0 时，`maxVolume` 或 `maxMacd` 为 0
- 导致 `horizontalInterval` 计算为 0，触发 FL Chart 的断言错误

**修复方案**:
- 在 `_buildVolumeChart()` 中添加 `safeMax` 变量，当 `maxVolume <= 0` 时使用 1.0
- 在 `_buildMacdChart()` 中添加 `safeMax` 变量，当 `maxMacd <= 0` 时使用 1.0
- 将 `safeMax` 用于 `maxY`、`minY` 和 `horizontalInterval` 的计算
