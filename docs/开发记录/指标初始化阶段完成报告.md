// 修改前：只取到 endIndex 的数据计算指标
final displayData = _allKlineData.take(endIndex).toList();

// 修改后：从历史起点计算完整指标
final fullData = _allKlineData.take(endIndex).toList();
final macdResult = IndicatorCalculator.calculateMACD(fullData);

// 构建完整结果（包含预热期的padding和真实指标值）
final macdOffset = fullData.length - macdResult.macd.length;
for (int i = 0; i < fullData.length; i++) {
  final macdIndex = i - macdOffset;
  if (macdIndex >= 0 && macdIndex < macdResult.macd.length) {
    result.add(MacdData(
      macd: macdResult.macd[macdIndex],
      diff: macdResult.dif[macdIndex],
      dea: macdResult.dea[macdIndex],
    ));
  } else {
    // 预热期：使用0值
    result.add(MacdData(macd: 0, diff: 0, dea: 0));
  }
}

## 4. 技术决策回顾

### 4.1 历史缓冲天数选择

**决策**：100天历史缓冲

**理由**：
- DMA指标需要50天预热期（最长）
- 100天缓冲 - 50天预热 = 50天有效历史指标
- 加上150天训练期 = 200天有效指标
- 数据量适中（250天），不会造成内存压力

### 4.2 预热期指标值处理

**决策**：预热期使用默认值（0或50）

**理由**：
- 预热期在历史区域（索引0-99），不影响训练期用户操作
- 用户主要关注训练期（100-249）指标
- 实现简单，不引入额外复杂性
- 专业交易软件通常也用0或不显示预热期数据

## 6. 已知问题与限制

### 6.1 当前限制

1. **预热期指标不准确**：技术指标的固有局限，EMA/SMA在预热期未收敛
2. **历史区域指标为0**：预热期显示为0或默认值，不影响训练期使用
3. **数据量限制**：如果数据库数据不足250天，可能无法全程显示有效指标

### 6.2 后续优化建议

1. **添加指标缓存**：避免每次 build 都重新计算指标
2. **预热期近似值**：使用简单移动平均作为 EMA 的近似，提供更平滑的预热期曲线
3. **指标计算优化**：使用 `compute()` 将计算移至后台线程
