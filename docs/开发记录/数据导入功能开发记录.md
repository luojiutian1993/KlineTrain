## 需求背景

用户需要将 `1000_stocks_post_fq_history` 上证股票历史数据导入到数据库，便于前端页面调取数据进行全流程验证。

---

## 实现方案

### 1. 创建Python导入脚本

**文件路径**: `scripts/import_stock_data.py`

**核心功能**:
- 读取Excel/CSV格式的股票数据文件
- 自动创建与drift兼容的数据库表结构
- 批量插入数据（每批5000条）
- 显示导入进度和速度

### 2. 数据库表结构

```sql
-- markets表
CREATE TABLE markets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    enabled INTEGER DEFAULT 1,
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- symbols表
CREATE TABLE symbols (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL UNIQUE,
    name TEXT,
    market_code TEXT NOT NULL,
    enabled INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- kline_data表
CREATE TABLE kline_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,
    market_code TEXT NOT NULL,
    period TEXT NOT NULL,
    trade_date TIMESTAMP NOT NULL,
    open REAL,
    close REAL,
    high REAL,
    low REAL,
    volume REAL,
    amount REAL,
    turnover_rate REAL,
    pe REAL,
    pb REAL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(symbol, period, trade_date)
);
```

### 3. 数据映射规则

| Excel/CSV列 | 数据库字段 | 说明 |
|-------------|-----------|------|
| code | symbol | 股票代码（如000001.XSHE） |
| date | trade_date | 交易日期 |
| open | open | 开盘价 |
| close | close | 收盘价 |
| high | high | 最高价 |
| low | low | 最低价 |
| volume | volume | 成交量 |

**市场识别规则**:
- 股票代码包含 `.XSHG` → 上海证券交易所 (XSHG)
- 股票代码包含 `.XSHE` → 深圳证券交易所 (XSHE)

## 遇到的问题及解决方案

### 问题1: CSV中部分数据为空

**现象**: CSV中有部分早期数据（2005-2019年）的open/close等字段为空

**解决方案**: 导入时过滤掉无效数据
```python
df_valid = df[df['open'].notna() & df['close'].notna()].copy()
```

### 问题2: 日期格式不统一

**现象**: CSV中日期格式可能是 `2015-01-01` 或 `2015/01/01`

**解决方案**: 兼容多种日期格式
```python
if isinstance(trade_date, str):
    try:
        trade_date = datetime.strptime(trade_date, '%Y-%m-%d')
    except:
        trade_date = datetime.strptime(trade_date, '%Y/%m/%d')
```
