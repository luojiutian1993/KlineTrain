## 数据库查询变更

### getKlineDataBefore 方法

原始查询：
```dart
return (select(klineData)
      ..where((t) => t.symbol.equals(symbol))
      ..where((t) => t.period.equals(period))
      ..where((t) => t.tradeDate.isSmallerOrEqualValue(date))
      ..orderBy([(t) => OrderingTerm.desc(t.tradeDate)])
      ..limit(days))
    .get();
```

修改后查询：
```dart
DateTime effectiveStartDate = startDate ?? DateTime(1990, 1, 1);
DateTime effectiveEndDate = endDate ?? date;

return (select(klineData)
      ..where((t) => t.symbol.equals(symbol))
      ..where((t) => t.period.equals(period))
      ..where((t) => t.tradeDate.isBiggerOrEqualValue(effectiveStartDate))
      ..where((t) => t.tradeDate.isSmallerOrEqualValue(effectiveEndDate))
      ..orderBy([(t) => OrderingTerm.desc(t.tradeDate)])
      ..limit(days))
    .get();
```

## UI交互流程

```
1. 用户进入首页选股区域
2. 用户看到时间范围选择 chips: [近1年] [近3年] [近5年] [自定义]
3. 用户点击预设选项 → 立即更新 timeRange 状态
4. 用户点击"自定义" → 弹出日期范围选择器
5. 用户选择日期范围 → 更新 timeRange 状态
6. 时间范围变化时，重新执行选股计算
```

## 测试验证

### 测试场景

1. **预设时间范围选择**
   - 选择"近1年" → timeRange.type = recent1Year
   - 选择"近3年" → timeRange.type = recent3Years
   - 选择"近5年" → timeRange.type = recent5Years

2. **自定义时间范围**
   - 点击"自定义" → 弹出日期选择器
   - 选择2015-01-01至2017-12-31 → 正确设置startDate和endDate

3. **选股计算验证**
   - 选择条件 + 时间范围 → 执行选股算法时传入时间范围参数
   - 检查K线查询是否限定在指定时间范围内
