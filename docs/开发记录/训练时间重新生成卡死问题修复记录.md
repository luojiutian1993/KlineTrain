# 训练时间重新生成卡死问题修复记录

**开发日期**: 2026-05-18
**版本**: v1.7.1
**开发者**: Claude AI

---

## 问题描述

用户反馈：选择训练时间的重新生成后，系统卡死无反应，页面长时间处于加载状态。

## 问题分析

通过日志分析和代码审查，发现两个严重的性能问题：

### 问题1：串行遍历性能瓶颈

`_filterSingleSymbolCheck` 方法串行遍历所有标的，对每个标的调用检查函数。当数据库中有大量标的（如1000+）时，每个标的都需要进行多次数据库查询，导致严重的性能问题。

**原代码**:
```dart
for (final symbol in allSymbols) {
  try {
    if (await check(symbol.symbol)) {
      results.add(symbol.symbol);
    }
  } catch (e) {
    continue;
  }
}
```

### 问题2：重复数据库查询

`_getStockDetails` 方法在循环中每次都调用 `getActiveSymbols()`，导致大量重复的数据库查询。

**原代码**:
```dart
for (final symbolCode in symbols) {
  final symbolList = await _stockFilterDao.getActiveSymbols();  // 每次循环都查询！
  final symbol = symbolList.firstWhere(...);
}
```

## 解决方案

### 1. 并行处理优化

将 `_filterSingleSymbolCheck` 方法改为并行处理：

```dart
final futures = allSymbols.map((symbol) async {
  try {
    if (await check(symbol.symbol)) {
      return symbol.symbol;
    }
  } catch (e) {
    // 忽略单个标的的错误
  }
  return null;
});

final results = await Future.wait(futures);
final validResults = results.where((s) => s != null).cast<String>().toList();
```

### 2. 避免重复查询

在 `_getStockDetails` 方法中一次性获取所有标的信息：

```dart
// 一次性获取所有标的信息，避免循环查询
final symbolList = await _stockFilterDao.getActiveSymbols();
final symbolMap = {for (var s in symbolList) s.symbol: s};

for (final symbolCode in symbols) {
  final symbol = symbolMap[symbolCode];  // O(1) 查找
  if (symbol == null) {
    continue;
  }
  // ...
}
```
