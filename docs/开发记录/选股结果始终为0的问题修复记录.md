## 问题描述

用户反馈：选择条件后（例如A股，上证&深证&上交所，随机时间为20230703，200日新高），未查询出结果，页面显示"请选择条件查询结果"。

## 问题分析

### 错误日志

```
Null check operator used on a null value
#0   $SymbolsTable.map (package:kline_trainer/data/database/app_database.g.dart:2991:70)
```

### 根本原因

发现了两个关键问题：

#### 问题1：状态管理不同步

存在两个独立的状态管理，彼此不同步：
- `selectionProvider` - 管理市场选择（`selectedSubMarkets`）
- `stockFilterProvider` - 管理筛选逻辑（`selectedMarkets`）

`MarketSelector` 更新的是 `selectionProvider`，但筛选使用的是 `stockFilterProvider`。

#### 问题2：数据库模型与实际表结构不匹配

Drift 模型中定义了 `lotSize` 和 `minTick` 字段，但实际数据库中的 `symbols` 表没有这些字段。当 Drift 尝试映射数据时，这些字段返回 null，但模型中使用了 `.withDefault()`，Drift 会强制解包这些值，导致 `Null check operator used on a null value` 错误。

**数据库实际表结构**:
```sql
CREATE TABLE symbols (
  id INTEGER PRIMARY KEY,
  symbol TEXT NOT NULL UNIQUE,
  name TEXT,
  market_code TEXT NOT NULL,
  enabled INTEGER DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Drift 模型定义**:
```dart
class Symbols extends Table {
  IntColumn get lotSize => integer().withDefault(const Constant(100))();
  RealColumn get minTick => real().withDefault(const Constant(0.01))();
  // ... 其他字段
}
```

## 解决方案

### 1. 修复数据库模型

将 `lotSize` 和 `minTick` 字段改为 nullable，避免强制解包空值：

```dart
class Symbols extends Table {
  /// 每手数量
  IntColumn get lotSize => integer().nullable()();

  /// 最小变动价位
  RealColumn get minTick => real().nullable()();
  
  // ... 其他字段
}
```

### 2. 同步状态管理

在 `selection_provider.dart` 中，让 `setSubMarkets` 方法调用 `_executeStockFilter()` 触发筛选：

```dart
void setSubMarkets(List<String> subMarkets) {
  state = state.copyWith(selectedSubMarkets: subMarkets);
  _executeStockFilter();  // 新增：触发筛选
}
```

在 `stock_condition_selector.dart` 中，让 `_onConditionSelected` 方法同步市场选择到 `stockFilterProvider`：

```dart
void _onConditionSelected(StockFilterCondition condition) {
  widget.onChanged(condition.label);
  
  // 获取 selectionProvider 中的市场选择
  final selectionState = ref.read(selectionProvider);
  final selectedSubMarkets = selectionState.selectedSubMarkets;
  
  // 同步市场选择到 stockFilterProvider
  if (selectedSubMarkets.isNotEmpty) {
    selectedSubMarkets.forEach((marketCode) {
      ref.read(stockFilterProvider.notifier).toggleMarket(marketCode);
    });
  }
  
  ref.read(stockFilterProvider.notifier).selectCondition(condition);
}
```
