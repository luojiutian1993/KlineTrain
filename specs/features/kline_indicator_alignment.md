# K线图表与技术指标对齐修复 - 需求文档

> 本文档记录 K线图表与技术指标垂直对齐修复功能的需求详情。

**文档版本**：v1.0.0
**创建日期**：2026-05-31
**最后更新**：2026-05-31
**需求来源**：用户反馈 - K线图表与技术指标错位问题

---

## 1. 背景与问题描述

### 1.1 现有问题

在实战页面和复盘页面中，K线图表的展示与技术指标未实现按日数据点一一垂直对齐。当前存在的问题是：

- 当用户对图表进行放大或缩小操作时，技术指标曲线与对应的K线数据点出现明显错位现象
- 副图指标（成交量、MACD、KDJ、RSI等）与K线主图的数据点无法精确对应
- 在不同缩放级别下，对齐问题更加明显

### 1.2 问题影响

- 用户无法准确判断特定日期对应的技术指标值
- 影响交易决策的准确性
- 降低了模拟训练的真实性体验

## 4. 技术分析

### 4.1 问题根因

当前实现中存在的问题：

1. **数据范围不一致**：K线主图和副图指标使用不同的数据范围计算
   - 主图使用 `displayKlineData`
   - 副图指标使用 `displayMacdData`、`displayVolumes` 等
   - 虽然都是根据 `visibleStartIndex` 和 `visibleKlineCount` 计算，但存在边界条件处理差异

2. **坐标系差异**：fl_chart 的 BarChart/LineChart 默认使用 `BarChartAlignment.spaceAround`，导致柱状/线条之间存在间距，无法与K线精确对齐

3. **渲染时机不同**：主图使用 CustomPainter 绘制，副图使用 fl_chart 组件，渲染逻辑独立

### 4.2 技术方案

采用**统一图表组件**方案：

1. **统一数据源**：确保K线和所有指标使用相同的数据范围计算逻辑
2. **统一坐标映射**：所有图表使用相同的 x 轴映射算法
3. **对齐参数传递**：从 BattleProvider 统一传递数据范围给所有图表组件

## 5. 验收标准

### 5.1 功能验收

| 序号 | 验收项 | 验收条件 |
|------|--------|----------|
| AC-001 | 初始状态对齐 | 页面加载后，K线第一个数据点与指标图第一个点垂直对齐 |
| AC-002 | 放大后对齐 | 放大图表（K线数量减少）后，第一个和最后一个数据点保持对齐 |
| AC-003 | 缩小后对齐 | 缩小图表（K线数量增加）后，所有数据点保持对齐 |
| AC-004 | 滑动后对齐 | 左右滑动图表后，数据点保持对齐 |
| AC-005 | 日K周期对齐 | 日K周期下，K线与指标精确对齐 |
| AC-006 | 周K周期对齐 | 周K周期下，K线与指标精确对齐 |
| AC-007 | 月K周期对齐 | 月K周期下，K线与指标精确对齐 |
| AC-008 | 复盘页面对齐 | 复盘页面同样实现精确对齐 |

### 5.2 测试场景

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 场景1 | 打开实战页面 | 主图K线与副图指标精确对齐 |
| 场景2 | 双指放大图表 | K线数量减少，指标同步减少，位置对齐 |
| 场景3 | 双指缩小图表 | K线数量增加，指标同步增加，位置对齐 |
| 场景4 | 左右滑动 | 数据同步滚动，位置对齐 |
| 场景5 | 切换周期 | 不同周期下均保持对齐 |
| 场景6 | 打开复盘页面 | 复盘页面同样精确对齐 |

## 6. 约束条件

1. **不影响现有功能**：修复对齐问题，但不能修改已完成的功能逻辑
2. **性能要求**：图表渲染流畅（60fps），不能因为对齐修复影响性能
3. **兼容性**：需要同时支持 iOS/Android 平台
