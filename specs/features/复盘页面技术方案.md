## 1. 技术概述

复盘功能基于现有实战页面（BattleScreen）实现，通过 `_isReplayMode` 标志位控制页面行为。核心改动包括路由传参、UI状态控制、买卖点渲染。

## 2. 架构设计

### 2.1 路由方案

**问题**: 当前 `records_screen.dart` 使用 `context.go('/battle', extra: {...})` 跳转到 `/battle` 路由，导致丢失底部导航栏（因为 `/battle` 使用 `_rootNavigatorKey`）。

**解决方案**: 修改 `/main` 路由支持复盘参数，当 `tab=1` 且带有复盘参数时，进入复盘模式。

```dart
// app_routes.dart
GoRoute(
  path: main,
  name: 'main',
  builder: (context, state) {
    final tabIndex = int.tryParse(state.uri.queryParameters['tab'] ?? '0') ?? 0;
    final mode = state.uri.queryParameters['mode'] ?? '';
    final sessionId = state.uri.queryParameters['sessionId'];
    
    BattleScreenParams? battleParams;
    
    if (tabIndex == 1) {
      final symbol = state.uri.queryParameters['symbol'] ?? '';
      final name = state.uri.queryParameters['name'] ?? '';
      final market = state.uri.queryParameters['market'] ?? '';
      final dateStr = state.uri.queryParameters['date'] ?? '';
      final DateTime? trainingStartDate = 
          dateStr.isNotEmpty ? DateTime.tryParse(dateStr) : null;
      
      battleParams = BattleScreenParams(
        symbol: symbol,
        name: name,
        market: market,
        trainingStartDate: trainingStartDate,
        isReplayMode: mode == 'replay',
        sessionId: sessionId != null ? int.tryParse(sessionId) : null,
      );
    }
    
    return MainTabPage(
      initialIndex: tabIndex,
      battleParams: battleParams,
    );
  },
),
```

→ 对应 AC-004（底部导航栏保持显示）

### 2.2 跳转链路

```
RecordsScreen._handleReplay()
        ↓
context.go('/main?tab=1&mode=replay&sessionId=xxx&symbol=...&market=...&date=...')
        ↓
MainTabPage 接收 battleParams
        ↓
BattleScreen 接收 battleParams
        ↓
BattleScreen 根据 battleParams.isReplayMode 初始化复盘模式
```

→ 对应 AC-001（加载历史数据）

## 4. 核心逻辑

### 4.1 复盘模式初始化

```dart
// battle_screen.dart
Future<void> _loadReplayMode(BattleScreenParams params) async {
  setState(() {
    _isReplayMode = true;
  });
  
  // 加载训练记录和交易数据
  final dbService = DatabaseService.instance;
  final session = await dbService.trainingDao.getSession(params.sessionId!);
  final trades = await dbService.trainingDao.getSessionTrades(params.sessionId!);
  
  if (session != null) {
    // 设置账户状态为训练结束时
    setState(() {
      _currentSymbol = session.symbol;
      _currentMarketCode = session.marketCode;
      _trainingStartDate = session.startDate;
      _trainingDays = session.endDate.difference(session.startDate).inDays + 1;
      _initialBalance = session.initialCapital;
      _accountBalance = session.currentCapital;
      _totalProfitLoss = session.totalProfit ?? 0;
    });
    
    // 加载K线数据
    await _loadKlineData();
    
    // 构建买卖点
    _buildTradePoints(trades);
    
    // 锁定到最后一天
    setState(() {
      _currentDayIndex = _allKlineData.length - 1;
      _trainingPhase = TrainingPhase.closing;
      _visibleStartIndex = (_currentDayIndex - _visibleKlineCount + 1)
          .clamp(0, _currentDayIndex);
    });
  }
}
```

## 9. 风险与注意事项

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 复盘数据缺失 | 无法加载历史交易记录 | 添加空数据检查和友好提示 |
| K线数据不足 | 训练周期短于预期 | 按实际数据量调整显示 |
| 性能问题 | 大量买卖点影响渲染 | 优化标记绘制逻辑 |
