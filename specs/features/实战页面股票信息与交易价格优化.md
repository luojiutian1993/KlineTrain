# 实战页面股票信息与交易价格优化 — 功能需求说明书

## 1. 背景与价值

**为什么做**：当前实战页面的股票信息展示逻辑不够贴近真实交易场景，用户在训练初期（开盘阶段）和训练后期（收盘阶段）看到的价格信息不一致，且交易价格没有根据训练阶段进行合理限制。

**做成什么样**：实现训练阶段感知的股票信息展示——开盘阶段显示开盘价并限制交易价格在当日高低价之间，收盘阶段显示收盘价并固定交易价格为收盘价。

**关联**：项目核心功能优化

---

## 2. 范围

### 本次做

- 初始化进入页面时，股票基本信息展示开盘价，显示"开"字标识
- 开盘阶段（第一次进入）计算涨跌差额和比例（基于前一天收盘价）
- 开盘阶段高、低、换手、量、金额显示"--"
- 开盘阶段买卖价格限制在当日最高价和最低价之间
- 第一次点击"下一步"后，股票信息变更为收盘价，显示"收"字标识
- 收盘阶段展示高、低、换手、量、金额（从数据库获取）
- 收盘阶段买卖价格固定为当日收盘价
- 第二次点击"下一步"进入下一天训练，循环执行

### 本次不做

- 换手、量、金额数据获取 — 待数据库更新后完善
- 条件单功能的完整实现 — 已在其他需求中处理
- 训练记录的数据库存储 — 已在其他需求中处理

## 4. 需求详述

### 4.1 核心流程

1. 用户进入实战页面 → 显示开盘阶段数据（开盘价、涨跌基于昨收、高/低等显示"--"）
2. 用户可在当日高低价限制内进行买卖操作
3. 用户点击"下一步" → 训练阶段切换为收盘状态，显示收盘价、高低价等完整数据
4. 用户可在收盘价固定价格进行买卖操作
5. 用户点击"下一步" → 进入下一天训练，回到步骤1

### 4.2 交互规则

- 训练阶段分为：开盘阶段（第1次进入）→ 收盘阶段（第1次点击下一步后）→ 下一天开盘（第2次点击下一步后）
- 买卖弹窗根据当前阶段自动设置价格范围或固定价格
- 股票信息区域实时反映当前训练阶段的数据状态

### 4.3 业务规则

| 编号 | 规则 |
|------|------|
| BR-001 | 初始化进入页面时，训练阶段为"开盘阶段" |
| BR-002 | 开盘阶段显示开盘价，价格标识为"开" |
| BR-003 | 开盘阶段涨跌额 = 开盘价 - 前一天收盘价 |
| BR-004 | 开盘阶段涨跌比例 = 涨跌额 / 前一天收盘价 × 100% |
| BR-005 | 开盘阶段高、低、换手、量、金额显示"--" |
| BR-006 | 开盘阶段买入价格必须在当日最低价 ≤ 价格 ≤ 当日最高价范围内 |
| BR-007 | 开盘阶段卖出价格必须在当日最低价 ≤ 价格 ≤ 当日最高价范围内 |
| BR-008 | 第一次点击"下一步"后，训练阶段切换为"收盘阶段" |
| BR-009 | 收盘阶段显示收盘价，价格标识为"收" |
| BR-010 | 收盘阶段涨跌额 = 收盘价 - 前一天收盘价 |
| BR-011 | 收盘阶段涨跌比例 = 涨跌额 / 前一天收盘价 × 100% |
| BR-012 | 收盘阶段显示当日最高价、最低价、换手率、成交量、成交额 |
| BR-013 | 收盘阶段买入价格固定为当日收盘价，不可修改 |
| BR-014 | 收盘阶段卖出价格固定为当日收盘价，不可修改 |
| BR-015 | 第二次点击"下一步"后，进入下一天训练，重置为开盘阶段 |

## 5. 验收标准

### 5.1 正常流程 (Happy Path)

- [ ] **AC-001**: 开盘阶段股票信息展示
  - **Given** 用户进入实战页面（训练初始化）
  - **When** 页面加载完成
  - **Then** 股票信息区域显示开盘价，价格后显示"开"字标识

- [ ] **AC-002**: 开盘阶段涨跌计算
  - **Given** 用户进入实战页面（训练初始化）
  - **When** 页面加载完成
  - **Then** 涨跌额和涨跌比例基于开盘价与前一天收盘价计算得出并正确显示

- [ ] **AC-003**: 开盘阶段未确定数据显示
  - **Given** 用户进入实战页面（训练初始化）
  - **When** 页面加载完成
  - **Then** 高、低、换手、量、金额字段显示"--"

- [ ] **AC-004**: 开盘阶段买入价格限制
  - **Given** 用户处于开盘阶段
  - **When** 点击买入按钮打开买入弹窗
  - **Then** 价格输入框默认显示开盘价，且输入值被限制在当日最低价和最高价之间

- [ ] **AC-005**: 开盘阶段卖出价格限制
  - **Given** 用户处于开盘阶段且有持仓
  - **When** 点击卖出按钮打开卖出弹窗
  - **Then** 价格输入框默认显示开盘价，且输入值被限制在当日最低价和最高价之间

- [ ] **AC-006**: 进入收盘阶段
  - **Given** 用户处于开盘阶段
  - **When** 点击"下一步"按钮
  - **Then** 训练阶段切换为收盘阶段，股票信息区域显示收盘价，价格后显示"收"字标识

- [ ] **AC-007**: 收盘阶段完整数据展示
  - **Given** 用户处于收盘阶段
  - **When** 页面更新完成
  - **Then** 显示当日最高价、最低价、换手率、成交量、成交额（换手率、成交量、成交额待数据库更新）

- [ ] **AC-009**: 收盘阶段买入价格固定
  - **Given** 用户处于收盘阶段
  - **When** 点击买入按钮打开买入弹窗
  - **Then** 价格输入框显示当日收盘价，且价格不可修改（只读状态）

- [ ] **AC-010**: 收盘阶段卖出价格固定
  - **Given** 用户处于收盘阶段且有持仓
  - **When** 点击卖出按钮打开卖出弹窗
  - **Then** 价格输入框显示当日收盘价，且价格不可修改（只读状态）

- [ ] **AC-011**: 进入下一天训练
  - **Given** 用户处于收盘阶段
  - **When** 点击"下一步"按钮
  - **Then** 训练进入下一天，页面重置为开盘阶段，显示新一天的开盘价

### 5.2 边界与异常 (Edge & Error Cases)

- [ ] **AC-012**: 开盘阶段价格超限提示
  - **Given** 用户处于开盘阶段，打开买入/卖出弹窗
  - **When** 用户输入价格低于最低价或高于最高价
  - **Then** 显示错误提示"价格超出当日波动范围"，输入无效

- [ ] **AC-013**: 收盘阶段无持仓卖出提示
  - **Given** 用户处于收盘阶段，无持仓
  - **When** 点击卖出按钮
  - **Then** 显示提示"没有持仓"，不打开卖出弹窗

- [ ] **AC-014**: 训练结束判断
  - **Given** 用户处于最后一天训练的收盘阶段
  - **When** 点击"下一步"按钮
  - **Then** 弹出训练结束对话框，不进入下一天

### 5.3 业务规则验证 (Business Rules)

- [ ] **AC-015**: 开盘阶段价格标识验证 ← BR-002
  - **Given** 用户进入实战页面（训练初始化）
  - **When** 查看股票信息区域
  - **Then** 价格后显示"开"字标识

- [ ] **AC-016**: 开盘阶段涨跌计算验证 ← BR-003, BR-004
  - **Given** 用户进入实战页面（训练初始化），已知前一天收盘价、当日开盘价
  - **When** 查看涨跌额和涨跌比例
  - **Then** 涨跌额 = 开盘价 - 前一天收盘价，涨跌比例计算正确

- [ ] **AC-017**: 开盘阶段未确定数据验证 ← BR-005
  - **Given** 用户进入实战页面（训练初始化）
  - **When** 查看股票信息区域
  - **Then** 高、低、换手、量、金额字段显示"--"

- [ ] **AC-018**: 开盘阶段价格范围验证 ← BR-006, BR-007
  - **Given** 用户处于开盘阶段，已知当日最高价和最低价
  - **When** 在买入/卖出弹窗输入价格
  - **Then** 价格低于最低价或高于最高价时无法提交

- [ ] **AC-019**: 收盘阶段价格标识验证 ← BR-009
  - **Given** 用户处于收盘阶段
  - **When** 查看股票信息区域
  - **Then** 价格后显示"收"字标识

- [ ] **AC-020**: 收盘阶段涨跌计算验证 ← BR-010, BR-011
  - **Given** 用户处于收盘阶段，已知前一天收盘价、当日收盘价
  - **When** 查看涨跌额和涨跌比例
  - **Then** 涨跌额 = 收盘价 - 前一天收盘价，涨跌比例计算正确

- [ ] **AC-021**: 收盘阶段价格固定验证 ← BR-013, BR-014
  - **Given** 用户处于收盘阶段
  - **When** 打开买入/卖出弹窗
  - **Then** 价格输入框为只读状态，显示当日收盘价

- [ ] **AC-022**: 训练阶段循环验证 ← BR-015
  - **Given** 用户处于收盘阶段
  - **When** 点击"下一步"按钮（非最后一天）
  - **Then** 进入下一天开盘阶段，显示新日期的开盘价
