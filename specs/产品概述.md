# 产品概述

## 1. 项目愿景 (Vision)
> 成为国内领先的K线模拟训练平台，帮助初级交易员通过历史真实K线训练提升盘感，降低真实交易的学习成本和风险。

## 2. 核心价值 (Value Proposition)

*   **用户价值**：
    *   **降低学习门槛**：无需真金白银即可进行真实市场环境的模拟训练
    *   **提升交易技能**：通过反复练习K线形态识别，提高对短期走势的判断能力
    *   **建立交易纪律**：通过条件单（止盈止损、网格交易）训练良好的风控习惯
    *   **数据驱动改进**：基于训练记录统计，清晰了解自身交易风格和短板
    *   **沉浸式体验**：平滑的K线走势和精准的买卖点位标记，提升训练真实感

*   **商业/技术价值**：
    *   **差异化竞争**：参考券商成熟的条件单模式，提供专业级模拟训练体验
    *   **用户粘性**：特殊指标和进阶功能规划（第二代产品），打造长期用户留存
    *   **技术可扩展**：Flutter跨平台架构，支持快速迭代和全平台覆盖
    *   **数据资产**：历史K线数据库为后续AI分析和策略回测奠定基础

## 3. 核心板块 (Core Modules)

### 3.1 首页模块
*   **描述**：提供选股条件、市场概览、推荐标的等功能，帮助用户快速找到训练目标
*   **关键特性**：
    *   选股条件筛选（行业、市值、技术指标等）
    *   时间范围选择（最近1年/3年/5年/自定义）
    *   市场热门标的推荐
    *   **确认按钮**：选择条件后点击确认跳转到实战页面
    *   今日训练统计概览

### 3.2 实战模块（增强版）
*   **描述**：核心训练模块，基于历史真实K线数据进行模拟交易训练，支持时间步进和精准点位标记
*   **关键特性**：
    *   **K线逐步揭示机制**：先开盘价 → 再完整K线，支持"下一步"按钮控制
    *   **平滑K线走势**：使用贝塞尔曲线优化均线渲染，视觉更流畅
    *   **买卖点位标记**：在K线图上精准标记用户的买入/卖出位置
    *   **时间步进控制**：支持播放/暂停、前进/后退、滑块拖动
    *   **训练周期显示**：下一步按钮前显示选股条件中选择的时间周期时长（如20天、60天）
    *   **换股功能**：训练中可切换标的，保留当前训练进度
    *   **自由买入/卖出交易**：弹出独立页面，支持选择价格、数量，提供仓位快捷选项
    *   **条件单系统**：止盈止损、条件买卖、网格交易
    *   **多周期切换**：日线/周线/月线
    *   **真实数据支撑**：从本地SQLite数据库读取历史K线数据
    *   **训练结果实时统计**

### 3.3 我的模块
*   **描述**：用户账户管理、个人训练记录和系统设置
*   **关键特性**：
    *   用户注册/登录（手机号+验证码）
    *   训练历史记录查询
    *   个人训练数据统计（胜率、盈亏比、最大回撤）
    *   账户与安全设置
    *   离线数据管理

## 4. 目标用户 & 核心场景 (User & Scenes)

### 4.1 目标用户

*   **[股票/期货新手]**：刚进入市场，缺乏实战经验，希望先模拟练习再入场
    *   核心诉求：低风险学习、熟悉K线形态、理解市场波动

*   **[有亏损经历的初级交易员]**：有一定交易经验但胜率不高，希望提升盘感
    *   核心诉求：找出交易短板、训练止损纪律、提高盈亏比

*   **[自我驱动的学习者]**：希望通过系统化训练形成自己的交易体系
    *   核心诉求：数据化复盘、量化自身进步、验证交易策略

### 4.2 核心场景 (Key Use Cases)

*   **场景 1**：[新手用户] 在通勤路上，打开APP选择"宁德时代"，系统从本地数据库加载2023年历史K线数据。用户通过点击"下一步"逐步揭示K线，在第30天价格回调时买入（图表标记红色三角形），在第45天达到止盈位后卖出（图表标记绿色三角形），最终查看本次训练的盈亏报告。

*   **场景 2**：[有经验的交易员] 在实战页面中，先点击"换股"选择"贵州茅台"，然后点击"下一步"推进K线。观察到均线形成金叉后，点击"买入"建仓。随后点击"条件单"设置止损位，继续点击"下一步"观察走势变化。

*   **场景 3**：[自我学习者] 使用时间步进器的播放功能自动推进K线，观察系统自动触发条件单的执行情况，中途通过滑块快速跳转到特定日期进行重点分析。

*   **场景 4**：[训练者] 在实战页面中，点击"下一步"按钮，K线从数据库读取下一天的真实数据并展示，账户盈亏实时更新。

*   **场景 5**：[用户] 在首页选择"上升趋势"条件和"最近60天"时间范围，点击确认按钮后跳转到实战页面，开始训练。

*   **场景 6**：[用户] 在实战页面点击买入按钮，弹出买入页面，选择全仓买入，点击确认完成买入操作。

## 5. 关键业务流程 (Core Workflows)

### 核心流程：K线训练会话（增强版）

```
1. [用户选择标的] → 系统展示可选品种列表（A股/期货）
        ↓
2. [系统加载历史数据] → 从本地SQLite数据库读取该品种历史K线数据
        ↓
3. [展示初始K线状态] → 显示训练起始日期和初始资金（默认10万）
        ↓
4. [训练循环开始]
   ┌─────────────────────────────────────────────────────────────────┐
   │  用户操作选择：                                                  │
   │  - 点击"下一步" → 推进到下一天                                   │
   │  - 点击"换股" → 切换标的，保留训练进度                            │
   │  - 点击"条件单" → 设置/修改/撤销条件单                            │
   │  - 点击"买入" → 弹出买入页面，设置价格和数量后确认买入             │
   │  - 点击"卖出" → 弹出卖出页面，设置价格和数量后确认卖出             │
   │         ↓                                                       │
   │  系统响应：                                                      │
   │  - 从数据库读取下一根K线数据                                      │
   │  - 平滑渲染均线（贝塞尔曲线）                                      │
   │  - 标记买卖点位（红色买入/绿色卖出三角形）                          │
   │  - 检查条件单触发并执行                                           │
   │  - 更新账户状态（资金、持仓、盈亏）                                │
   └─────────────────────────────────────────────────────────────────┘
        ↓
5. [训练结束条件触发]
   - K线走到数据末尾
   - 用户主动结束训练
   - 资金耗尽或亏损达50%
        ↓
6. [生成训练报告]
   - 总交易次数、胜率
   - 总盈亏金额、盈亏比
   - 最大单次盈利/亏损
   - 最大回撤
   - 条件单触发统计
```

### 首页选股流程

```
用户进入首页
        ↓
选择时间范围（最近1年/3年/5年/自定义）
        ↓
选择选股条件（趋势向上/趋势向下）
        ↓
点击"确认"按钮
        ↓
跳转到实战页面，携带筛选条件
        ↓
根据条件加载股票列表或直接开始训练
```

### 买入流程

```
用户点击"买入"按钮
        ↓
弹出买入页面
        ↓
设置买入价格（默认当前价）
        ↓
设置买入数量或选择仓位（1/3仓、1/2仓、全仓）
        ↓
点击"确认" → 完成买入，返回到实战页面
   或
点击"取消" → 返回到实战页面，不执行买入
```

### 卖出流程

```
用户点击"卖出"按钮
        ↓
弹出卖出页面
        ↓
设置卖出价格（默认当前价）
        ↓
设置卖出数量或选择仓位（1/3仓、1/2仓、全仓）
        ↓
点击"确认" → 完成卖出，返回到实战页面
   或
点击"取消" → 返回到实战页面，不执行卖出
```

## 6. 验收标准 (Acceptance Criteria)

*   **功能完备性**：
    *   用户可完成完整的模拟训练流程（选择标的 → 揭示K线 → 交易 → 查看报告）
    *   "下一步"按钮正常工作，每次点击推进一天K线
    *   K线数据从本地SQLite数据库读取，非模拟数据
    *   换股、条件单、买入、卖出模块均支持"下一步"操作
    *   买卖点位在K线图上正确标记（红色买入/绿色卖出三角形）
    *   均线使用平滑曲线渲染，视觉流畅
    *   时间步进器（播放/暂停、前进/后退、滑块）正常工作
    *   条件单系统可正常工作（止盈止损、条件买卖、网格交易）
    *   账户系统可用（注册、登录、保存训练记录）
    *   首页选股条件确认按钮正常工作，点击后跳转到实战页面
    *   实战页面显示训练周期时长（如20天、60天）
    *   买入页面支持设置价格、数量和仓位快捷选项
    *   卖出页面支持设置价格、数量和仓位快捷选项
    *   无崩溃、卡死等严重Bug

*   **技术指标**：
    *   K线图表渲染流畅（60fps）
    *   页面加载时间 < 2秒
    *   数据库查询响应时间 < 100ms
    *   K线切换动画平滑，无卡顿
    *   iOS端可通过App Store审核上架

*   **合规要求**：
    *   应用内显著位置显示免责声明
    *   明确标注"模拟训练，不构成投资建议"
    *   不涉及任何真实证券交易

## 7. 约束条件 (Constraints)

*   **平台约束**：第一期仅开发iOS端（Flutter架构便于后续扩展Android/Mac/Windows）
*   **数据约束**：K线数据从本地SQLite数据库读取，确保离线可用
*   **账户约束**：必须注册登录，游客不可使用任何功能
*   **合规约束**：产品定位为教育工具，非证券交易软件，不需相关金融牌照
*   **开发周期**：MVP版本控制在10周内完成
*   **性能约束**：K线渲染必须流畅，支持至少1000根K线数据展示

## 8. 项目边界 (Out of Scope)

*   **暂不支持的功能**：
    *   真实交易（与券商对接）
    *   实时行情数据
    *   资讯/新闻/基本面数据
    *   自选股同步
    *   社交功能（排行榜、对战）
    *   社区/论坛
    *   AI辅助分析（第二代产品规划）
    *   策略回测功能（第二代产品规划）

*   **暂不支持的平台**：
    *   Android（第二期规划）
    *   Mac桌面端（第三期规划）
    *   Windows桌面端（第三期规划）
    *   Web端

*   **暂不支持的市场**：
    *   港股、美股（第一期仅支持A股和商品期货）

## 9. 新增功能详细设计

### 9.1 平滑K线走势

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 均线平滑 | 使用贝塞尔曲线渲染均线，替代传统折线 | CustomPaint + Path.quadraticBezierTo |
| K线渲染优化 | 优化Canvas绘制逻辑，提高渲染效率 | isAntiAlias=true, 批量绘制 |
| 动画过渡 | K线切换时添加平滑过渡动画 | AnimatedBuilder + Tween |

### 9.2 买卖点位标记

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 买入标记 | 在K线图上标记买入位置（红色三角形向上） | CustomPaint绘制三角形路径 |
| 卖出标记 | 在K线图上标记卖出位置（绿色三角形向下） | CustomPaint绘制三角形路径 |
| 标签显示 | 在标记旁显示交易价格和日期 | TextPainter绘制文字 |
| 交互提示 | 点击标记显示详细交易信息 | GestureDetector + Tooltip |

### 9.3 按钮交互说明

| 按钮 | 功能 | 行为 |
|------|------|------|
| **下一步** | 时间推进 | 点击后K线推进一天，从数据库加载下一天的真实K线数据，同时更新账户状态（持仓、盈亏、资金） |
| **换股** | 切换标的 | 弹出选股页面，根据前置的选股条件筛选并选择新的股票标的，切换后从新的股票历史数据继续训练 |
| **条件单** | 自动交易 | 弹出条件单设置页面，用户设置买卖条件（价格条件、时间条件、技术指标条件），满足条件时自动触发买入/卖出 |
| **买入** | 市价买入 | 弹出买入页面，支持设置买入价格、买入数量，提供1/3仓、1/2仓、全仓快捷选项，点击确认完成买入 |
| **卖出** | 市价卖出 | 弹出卖出页面，支持设置卖出价格、卖出数量，提供1/3仓、1/2仓、全仓快捷选项，点击确认完成卖出 |

### 9.4 时间步进器

| 控件 | 功能 | 行为 |
|------|------|------|
| 后退按钮 | 后退一天 | 当前天数-1，显示上一天K线 |
| 播放/暂停按钮 | 自动播放/暂停 | 每2秒自动推进一天 |
| 前进按钮 | 前进一天 | 当前天数+1，显示下一天K线 |
| 滑块 | 快速定位 | 拖动滑块跳转到指定日期 |
| 日期显示 | 当前日期 | 显示当前显示的K线日期 |
| 进度显示 | 训练进度 | 显示"当前天数/总天数" |
| 周期显示 | 训练周期时长 | 显示选股条件中选择的时间周期对应的时长（如20天、60天） |

### 9.5 数据库数据读取

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 数据优先 | 优先从数据库读取K线数据 | Drift ORM查询 |
| 数据回退 | 数据库无数据时使用API | Dio网络请求 |
| 数据缓存 | API数据存入数据库 | Batch插入 |
| 数据格式 | 统一数据模型 | KlineModel实体类 |

### 9.6 买入页面设计

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 价格输入 | 支持手动输入买入价格，默认显示当前K线收盘价 | TextField + InputDecoration |
| 数量输入 | 支持手动输入买入数量 | TextField + TextInputType.number |
| 仓位快捷选项 | 提供1/3仓、1/2仓、全仓快捷按钮 | Row + ElevatedButton |
| 买入金额预览 | 实时显示买入所需金额 | Text + 状态监听 |
| 确认按钮 | 点击后执行买入并返回实战页面 | Navigator.pop + 状态更新 |
| 取消按钮 | 点击后返回实战页面，不执行买入 | Navigator.pop |

### 9.7 卖出页面设计

| 功能点 | 描述 | 技术实现 |

### 9.8 首页确认按钮设计

| 功能点 | 描述 | 技术实现 |
|--------|------|----------|
| 确认按钮 | 在选股条件区域底部显示，点击后跳转到实战页面 | ElevatedButton + Navigator.push |
