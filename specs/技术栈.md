## 3. 数据库技术决策记录

### 3.1 ORM选择: Drift

| 选项 | 评估 | 决策 |
|------|------|------|
| sqflite | 原始SQL，易用但无类型安全 | ❌ |
| Moor (Drift旧版) | 成熟稳定，但已更名 | ❌ |
| Drift | 类型安全，代码生成，跨平台支持 | ✅ 采用 |

**理由**: Drift提供类型安全的SQL操作，自动生成代码，支持所有Flutter平台（iOS/Android/macOS/Windows），内置迁移支持。

### 3.2 本地数据库选择: SQLite

| 选项 | 评估 | 决策 |
|------|------|------|
| SQLite | 内置、轻量、跨平台、无需服务 | ✅ 采用 |
| Hive | 快但不支持关系查询 | ❌ |
| ObjectBox | 轻量但生态小 | ❌ |

**理由**: SQLite成熟稳定，内置在所有移动平台，关系模型适合业务需求。

### 3.3 云端数据库选择: 阿里云RDS

| 选项 | 评估 | 决策 |
|------|------|------|
| 阿里云RDS MySQL | 高可用、自动扩容、国内访问快 | ✅ 采用 |
| AWS RDS | 功能强但国内访问慢 | ❌ |
| Google Cloud SQL | 功能强但国内访问慢 | ❌ |

**理由**: 国内项目优先选择阿里云生态，集成简单，成本可控。

## 4. 技术决策记录

### 4.1 图表库选择

| 选项 | 评估 | 决策 |
|------|------|------|
| Syncfusion Charts | 功能强大但与Flutter 3.41存在API冲突 | ❌ 放弃 |
| fl_chart | 轻量、稳定、兼容性好 | ✅ 采用 |

**理由**: Syncfusion与Flutter 3.41的ThemeData API存在命名冲突（`SelectionDetails`），而fl_chart更轻量且没有兼容性问题。

### 4.2 ThemeData API适配

Flutter 3.17+ 对ThemeData API进行了调整：

| 旧API | 新API |
|-------|-------|
| `accentColor` | `colorScheme.secondary` |
| `backgroundColor` | `colorScheme.background` |
| `FontWeight.semiBold` | `FontWeight.w600` |

## 6. 运行注意事项

### 6.1 macOS构建

首次运行macOS需要：
1. 添加macOS平台支持：`flutter create . --platforms macos`
2. 安装CocoaPods：`brew install cocoapods`
3. 禁用GPU着色器编译（解决编译问题）：
   ```bash
   FLUTTER_DISABLE_GPU_SHADERS=true flutter run -d macos --release
   ```
