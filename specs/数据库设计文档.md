## 1. 数据库方案概览

### 1.1 部署策略

采用 **Phase 1 本地 → Phase 2 阿里云** 的渐进式迁移策略：

```
┌─────────────────────────────────────────────────────────────┐
│                    Phase 1: 本地开发阶段                      │
├─────────────────────────────────────────────────────────────┤
│  技术栈: SQLite (Drift)                                     │
│  优势:                                                     │
│    - 无需网络，随时可开发                                   │
│    - 单用户场景性能优秀                                    │
│    - 开发调试简单                                         │
│  适用: MVP阶段，本地开发、测试                              │
└─────────────────────────────────────────────────────────────┘
                            ↓ 平稳迁移
┌─────────────────────────────────────────────────────────────┐
│                    Phase 2: 阿里云部署阶段                   │
├─────────────────────────────────────────────────────────────┤
│  技术栈:                                                   │
│    - 阿里云RDS MySQL 8.0 (主数据库)                        │
│    - 阿里云Redis (缓存/会话)                                │
│    - 阿里云OSS (K线历史文件存储)                            │
│    - 阿里云表格OTS (时序K线数据)                           │
│    - 阿里云MaxCompute (大数据分析)                         │
│  优势:                                                     │
│    - 高可用、自动备份                                       │
│    - 弹性扩展                                              │
│    - 支持多用户并发                                        │
│    - 大数据分析能力                                        │
│  适用: 正式上线运营                                        │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2.1 用户表 (users)

```dart
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get phone => text().unique().withLength(min: 11, max: 11)();
  TextColumn get password => text()(); // 加密存储 (SHA-256 + Salt)
  TextColumn get nickname => text().nullable()();
  TextColumn get avatar => text().nullable()(); // 头像URL
  TextColumn get email => text().nullable()();
  IntColumn get memberLevel => integer().withDefault(const Constant(1))(); // 会员等级 1-10
  IntColumn get experience => integer().withDefault(const Constant(0))(); // 经验值
  RealColumn get totalProfit => real().withDefault(const Constant(0.0))(); // 累计盈亏
  IntColumn get totalTrainingDays => integer().withDefault(const Constant(0))(); // 累计训练天数
  RealColumn get avgDailyDuration => real().withDefault(const Constant(0.0))(); // 日均训练时长(分钟)
  RealColumn get overallWinRate => real().withDefault(const Constant(0.0))(); // 总胜率
  RealColumn get avgProfitRate => real().withDefault(const Constant(0.0))(); // 平均收益率
  TextColumn get status => text().withDefault(const Constant('active'))(); // active/inactive/banned
  DateTimeColumn get lastLoginAt => dateTime().nullable()();
  DateTimeColumn get createdAt => dateTime().withDefault(currentDateAndTime)();
  DateTimeColumn get updatedAt => dateTime().withDefault(currentDateAndTime)();
}
```

**索引**:
- `phone` (唯一索引) - 登录查询
- `memberLevel` (普通索引) - 会员等级筛选
- `status` (普通索引) - 用户状态筛选

- 5年 × 250交易日 × 5000标的 ≈ 6250万条（日K）
- 5年 × 240交易日 × 4小时 × 12时段 × 1000标的 ≈ 5.76亿条（5分钟K）

### 4.2 高收益率用户特征挖掘

```sql
-- 高收益率用户特征分析SQL示例
SELECT
  user_id,
  AVG(profit_rate) as avg_profit_rate,
  AVG(win_rate) as avg_win_rate,
  AVG(profit_loss_ratio) as avg_pl_ratio,
  AVG(max_drawdown) as avg_drawdown,
  COUNT(DISTINCT session_id) as training_count,
  SUM(trade_count) as total_trades,
  -- 时间特征
  HOUR(AVG(created_at)) as peak_trading_hour,
  -- 市场偏好
  MODE(market_code) as preferred_market,
  -- 周期偏好
  MODE(period) as preferred_period,
  -- 策略特征
  AVG(condition_order_count) as avg_condition_orders,
  AVG(condition_order_triggered / condition_order_count) as condition_effectiveness
FROM training_sessions
WHERE profit_rate > 30  -- 高收益率筛选
  AND trade_count >= 10  -- 有效样本量
GROUP BY user_id
HAVING COUNT(DISTINCT session_id) >= 5  -- 至少5次训练
ORDER BY avg_profit_rate DESC
LIMIT 100;
```
